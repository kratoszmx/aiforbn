AI for BN PoC — Current Workflow Brief
======================================

Purpose
-------
This file is no longer an MVP start plan. It is a compact snapshot of the repo's current workflow state so future coding rounds do not fall back to the old "baseline + demo UI" framing.

Current project goal
--------------------
Build a PhD-level AI for BN research PoC that is:
1) runnable,
2) methodologically honest,
3) explicit about the difference between overall evaluation and formula-only screening,
4) able to bridge ranked BN-themed formula candidates into a concrete structure-follow-up workflow.

Current stable mainline
-----------------------
The last fully verified mainline already includes:
- public 2D materials data ingestion and normalized caching
- grouped-by-formula evaluation
- grouped robustness benchmarking
- BN formula holdout benchmark
- BN family holdout benchmark
- BN vs non-BN stratified error benchmarking
- candidate-compatible honesty tables
- ranking uncertainty / abstention / decision-policy layers
- BN-centered alternative ranking
- structure-generation handoff and first-pass execution artifacts

Current default modeling surface
--------------------------------
Feature families kept in the mainline:
- `basic_formula_composition`
- `matminer_composition`
- `fractional_composition_vector`
- `matminer_composition_plus_structure_summary`

Default model sweep kept in the mainline:
- `linear_regression`
- `hist_gradient_boosting`
- `torch_mlp`
- `torch_mlp_ensemble`

Important interpretation:
- overall best evaluation model and best candidate-compatible screening model may differ
- the strongest currently verified candidate-compatible neural control is still:
  - `matminer_composition + torch_mlp_ensemble`

Current experimental branch status
----------------------------------
The live experimental tree has added pilot-only composition models:
- `torch_fractional_attention`
- `torch_sparse_fractional_attention`
- `torch_roost_like`

Current short-pilot conclusions:
- dense fractional attention: negative
- sparse fractional attention: negative / unstable
- Roost-like: mildly interesting, but still below dummy on BN-slice
- kNN sanity check: clearly not the next route

Therefore:
- these experimental models should remain out of the default sweep
- the scientific mainline conclusion is not yet changed by them

Current blocked next step
-------------------------
The most plausible next small-tabular modern baseline identified so far is `TabPFN`.

Current status:
- `tabpfn==7.1.1` is installed in `quant`
- actual pilot execution is blocked by Prior Labs license acceptance plus missing `TABPFN_TOKEN`
- this is an auth/license blocker, not a compute blocker

Workflow rule for future modeling rounds
----------------------------------------
1. Re-read skills + `HANDOFF.md`
2. Preserve the last stable mainline as the fallback truth
3. Test new model ideas only through short local pilots first
4. Promote nothing into the default sweep without stronger BN evidence
5. If a future model shows strong signal and then becomes compute-limited, ask for the user's ready GPU machine

Current honest claim boundary
-----------------------------
What the repo can support now:
- a stronger and more honest research PoC than the original MVP
- clear BN-centered diagnostics
- a defensible candidate-ranking workflow with follow-up structure handoff

What it still cannot support:
- strong BN-centered generalization claims
- structure validation or discovery claims
- the statement that the latest experimental branch has already found a clearly better modern-model replacement for the current mainline
