# AI for BN 项目简报（初稿）

## 1. 这是什么项目

这是一个面向“BN 相关材料”的小型 AI 演示项目。

这里的 BN 指的是 **硼氮化物**，英文是 **boron nitride**。更广一点说，这个项目现在关注的是“**含有 B（硼）和 N（氮）元素的材料，以及和 BN 思路相近的一组候选材料**”。

这个项目目前不是一个完整科研成果，更准确地说，它是一个 **PoC（Proof of Concept，概念验证）**：

- 目标不是一下子做出最强模型
- 目标是先证明“这条路线能跑通”
- 也就是先把“数据获取 -> 材料筛选 -> 特征构造 -> 模型训练 -> 候选打分 -> 结果展示”这一整条流程搭起来

如果以后要扩展成课程项目、论文前期工作、组会展示材料，这个版本可以作为一个清晰的出发点。

## 2. 为什么主题是 BN

选择 BN 作为主题，主要有三个原因。

第一，**BN 是一个比较有代表性的材料主题**。在材料科学里，BN 相关材料经常出现在绝缘、半导体、二维材料等讨论里，既不是完全冷门，也足够有研究价值。

第二，**BN 很适合做一个教学型 AI demo**。因为它背后有一个很自然的问题：

- 已知一些含 B、N 的材料数据
- 能不能用机器学习先学出一个粗略规律
- 再去筛一批“可能值得继续关注”的候选材料

这个问题对老师和同学都比较容易理解，不需要先讲很复杂的化学背景。

第三，**BN 主题容易向后扩展**。当前项目只是一个第一版，但以后可以继续往下做：

- 从纯 BN 扩展到更多 III-V 族材料
- 从简单组成特征扩展到结构特征
- 从演示级筛选扩展到更严谨的候选发现流程

## 3. 当前 demo 已经能做什么

现在这个 demo 已经可以跑通一条完整的最小流程。

### 3.1 自动拿到一份公开材料数据

项目会从公开数据源中读取一份二维材料数据集（当前配置是 `twod_matpd`，通过 JARVIS-Tools 获取）。

你可以把它理解为：先拿到一张“材料样本表”，里面每一行是一种材料，带有一些基础信息。

### 3.2 找出 BN 相关样本

项目会从数据里筛出化学式中同时包含 **B** 和 **N** 的记录，也就是先提取一个“BN 相关子集”。

这一步的作用是：

- 让主题聚焦在 BN
- 给后续展示一个更容易解释的切片结果

### 3.3 用很基础的方法把材料变成模型能读的数据

机器学习模型不能直接“看懂”化学式，所以项目会先把化学式转成一些简单数字特征。

例如：

- 这个材料里有几个元素
- 元素原子序数的大致和、均值、最大值、最小值
- 是否包含 B
- 是否包含 N

这类特征并不高级，但它的好处是：

- 容易实现
- 容易解释
- 适合做第一版 demo

### 3.4 训练一个基线模型来预测 band gap

当前项目的目标性质是 **band gap（带隙）**。

如果用非常直白的话来解释，带隙可以理解为：**电子要跨过的一道“能量门槛”**。它和材料更像绝缘体、半导体还是更容易导电，有很强关系。

项目现在会训练一个 sklearn 回归模型，去学习：

- 输入：材料的简单组成特征
- 输出：该材料的 band gap

这不是最先进的方法，但足够完成一个“AI 能否初步参与材料筛选”的演示。

### 3.5 生成一批 BN 风格候选材料并做打分

项目会自动生成一组简单候选，比如把 B/N 对应到同族元素组合，形成一批“BN 风格”材料名单。

然后模型会对这些候选材料做预测，并输出一个按预测值排序的 Top-K 结果。

可以把这一步理解为：

- 不是让 AI 直接宣布“新材料被发现了”
- 而是让 AI 先给出一个“值得优先关注的候选名单”

### 3.6 输出可以展示的结果文件和图

程序运行后会在 `artifacts/` 下生成一些结果，例如：

- `metrics.json`：模型指标
- `predictions.csv`：测试集预测结果
- `bn_slice.csv`：BN 相关样本切片
- `screened_candidates.csv`：候选筛选结果
- `parity_plot.png`：真实值和预测值的对比图

这些文件已经足够支撑一个基础汇报或课堂演示。

## 4. 如何运行和演示

下面按“最容易给老师演示”的方式来讲。

### 4.1 命令行跑主流程

在项目根目录执行：

```bash
cd ~/projects/ai_for_bn
conda activate quant
python main.py
```

如果环境里没有激活 `quant`，也可以直接使用当前项目已经验证过的解释器路径：

```bash
/opt/homebrew/Caskroom/miniforge/base/envs/quant/bin/python main.py
```

运行成功后，终端会打印：

- 数据集行数
- BN 子集行数
- 候选材料行数
- 模型评估指标

同时结果会写入 `artifacts/`。

### 4.2 用 Notebook 分步演示

如果想边讲边演示，可以打开项目根目录下的 `main.ipynb`。

这个 notebook 现在已经适配了 Jupyter 的运行方式，不再依赖 `__file__`，因此更适合课堂或组会现场一步一步执行。

### 4.3 用 Streamlit 做一个更直观的展示

在已经运行过 `main.py` 之后，可以执行：

```bash
streamlit run apps/streamlit_app.py
```

页面会显示：

- 指标结果
- 预测样例
- 候选材料筛选结果

如果老师更偏向“看结果页面而不是看代码”，这一步会比较有帮助。

## 5. 向老师介绍时可以怎么说

如果要用很简洁的话介绍，这个项目可以概括成下面这几句：

1. 这是一个面向 BN 相关材料的 AI 概念验证项目。
2. 它已经能把公开数据、简单特征、基线模型和候选筛选串成一条完整流程。
3. 目前重点不是追求最强精度，而是先验证“AI 辅助材料初筛”这件事能否顺畅落地。
4. 下一步可以在特征、模型、候选生成和实验关联性上继续加强。

## 6. 当前局限

这个项目现在是一个“能跑、能讲、能展示”的第一版，但离科研级系统还有明显距离。

### 6.1 特征比较简单

现在主要用的是从化学式直接提取出来的基础组成特征。

这意味着模型看到的是一种“简化后的材料信息”，还没有真正利用更丰富的晶体结构、电子结构或更专业的材料描述符。

### 6.2 候选生成规则比较粗

当前候选材料更多是“按元素替换规则生成的一批演示名单”，它的作用是支持 demo，而不是保证每个候选都化学上合理、稳定、可合成。

### 6.3 评估仍然偏 PoC 风格

虽然有训练、测试和指标输出，但这还不等于严格的科研验证。

后续如果要更严谨，需要考虑：

- 更合理的数据切分
- 更稳定的交叉验证
- 与已有文献或基线方法做对比

### 6.4 还没有实验闭环

目前结果主要是“模型预测与排序”，还没有和实验设计、第一性原理计算或文献验证形成闭环。

所以当前输出应该被理解为“**值得优先查看的线索**”，而不是最终结论。

## 7. 下一步建议

下面这些方向比较现实，也适合作为后续汇报中的“下一阶段计划”。

### 7.1 先把 demo 做得更稳

- 固定运行环境
- 补充更清晰的结果说明
- 让 notebook、命令行、网页展示三条路径都更顺手

这是最容易在短时间内提升展示效果的一步。

### 7.2 提升特征工程

可以逐步加入更有材料意义的特征，例如：

- 更丰富的元素统计特征
- 结构相关特征
- 现成材料特征工具生成的描述符

这通常比一开始盲目换更复杂模型更有效。

### 7.3 让候选生成更可信

后续可以把候选生成从“演示型枚举”升级成“带有材料常识约束的筛选”。

例如：

- 加入价态或稳定性相关规则
- 结合已知材料库去重
- 对高分候选再做更严格的二次判断

### 7.4 提高汇报的科研味道

如果准备给老师或教授做更正式的汇报，可以补上这些内容：

- 为什么 band gap 是合理的第一目标
- BN 与相关 III-V 材料的背景介绍
- 当前方法相比传统人工筛选的潜在价值
- 这个 PoC 下一步如何接近真正研究问题

## 8. 总结

一句话总结：**这个项目已经完成了一个“面向 BN 相关材料的 AI 初筛 demo”第一版。**

它的价值不在于“已经找到最优材料”，而在于：

- 主题明确
- 流程完整
- 结果可展示
- 后续扩展方向清楚

对于当前阶段来说，这样的状态是适合作为课程项目、组会展示或论文前期思路汇报的。
