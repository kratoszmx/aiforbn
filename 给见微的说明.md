# 给见微的说明

## 一句话结论

这个项目现在最准确的定位是：

**一个已经从“能跑的 band-gap demo”升级成“方法学更诚实、BN 主题更明确、并且开始把候选排序桥接到后续结构工作流”的研究型 PoC。**

它已经明显比早期版本更像一个可以拿去和导师讨论的博士项目雏形，但还没有到“可以直接主张发现新材料”的阶段。

---

## 1. 现在这个项目到底做到了什么

如果不看代码、只看你最需要知道的结论，可以概括成四件事：

1. **评估更可信了**
   - 主评估继续使用按 `formula` 分组切分，避免同一公式泄漏到 train/val/test。
   - 现在还额外做了 `grouped-by-formula` 的 5-fold robustness benchmark，不再只依赖一次 holdout 结果。
   - 又专门加了 **BN-slice leave-one-BN-formula-out** 诊断，专门检查 BN 方向到底有没有真的泛化。

2. **BN 终于更“在项目中心”了**
   - 候选空间已经从旧的 generic toy grid 改成 **BN-anchored formula-family grid**。
   - 当前候选生成围绕 `BN / BCN / BC2N / Si2BN` 一类 BN 相关母题组织，不再像以前那样只是给 generic 候选贴 BN 标签。

3. **候选筛选不再只是一个 top-k 分数表**
   - 现在会同时输出：
     - raw ranking
     - proposal shortlist
     - extrapolation shortlist
     - BN-centered alternative ranking
   - 同时加入 `BN-local support`、`BN analog evidence`、`BN analog validation penalty`、`grouped robustness penalty` 等解释层。

4. **项目已经开始回答“排完名之后下一步做什么”**
   - 现在不仅有候选排名，还有 structure seed、handoff JSON、reference records、job plan、first-pass queue、prototype-grounded follow-up shortlist。
   - 也就是说，系统已经把公式排序和后续结构工作之间，搭了一座真正可执行的桥。

---

## 2. 当前最重要的结果应该怎么理解

### 2.1 一般二维材料 band-gap 预测这件事，当前系统已经做得像样了

当前主 benchmark 下：

- 最优 overall evaluation 组合：
  - `matminer_composition_plus_structure_summary + hist_gradient_boosting`
- test 指标：
  - `MAE = 0.5568`
  - `RMSE = 0.8218`
  - `R² = 0.6302`

最佳 formula-only 路径为：

- `matminer_composition + hist_gradient_boosting`
- test `MAE = 0.5822`

这意味着：

- 在一般二维材料 band-gap 预测上，当前轻量 structure-aware 路径相对最佳 composition-only 路径，确实有**小幅但一致**的提升。
- 这件事不仅出现在单次 test split 上，在 grouped-by-formula 的 5-fold robustness benchmark 里也成立。

所以如果只问“这个项目是不是已经比一个普通 baseline demo 强很多”，答案是**是的**。

### 2.2 但 BN-centered generalization 仍然是主要短板

这也是现在最应该诚实说明的地方。

当前新增的 BN-slice benchmark 会把每个 BN 公式轮流留出，专门看 BN 方向的外推。结果显示：

- selected model：`MAE = 1.6397`
- screening model：`MAE = 1.6383`
- global dummy baseline：`MAE = 1.3257`
- 当前 BN-slice 上最好的组合：
  - `matminer_composition_plus_structure_summary + linear_regression`
  - `MAE = 1.2889`

这代表：

- 当前整体最优模型和当前筛选模型，在 BN-slice 上都**没有打赢**最简单的 dummy baseline；
- 只有一个 structure-aware + linear 的组合在 BN-slice 上略微优于 dummy。

所以现在最准确的话术不是：

- “我们已经把 BN 做出来了”

而是：

- “我们已经把 BN 方向的问题单独拆出来认真测了，而且测出来它目前仍然是主要短板。”

这其实很重要，因为它让项目从“看起来有很多东西”变成“知道自己哪里真的还不够”。

---

## 3. 现在比旧版本真正强在哪里

如果从“给导师看是否更像博士项目”这个角度讲，我认为当前版本主要强在下面三点。

### 3.1 不再混淆不同问题

现在项目会明确区分：

- **overall evaluation best**
- **formula-only screening best**
- **BN-slice best**

这三个角色现在已经不是一个东西。

这非常重要，因为以前最容易被质疑的地方之一，就是把“整体 benchmark 最优”偷偷当成“候选筛选最优”甚至“BN 方向也最优”。

现在这个混淆基本已经被拆开了。

### 3.2 不再只会给一个 top-k

现在系统不会只说“前 10 个候选是哪些”，而是会分层给出：

- 完整 raw ranking
- family-aware proposal shortlist
- 只保留真正 unseen formulas 的 extrapolation shortlist
- BN-centered alternative ranking
- prototype-grounded structure follow-up shortlist
- novelty-aware structure follow-up shortlist

这会让你和导师讨论时，不再只能围绕“这个公式分数高不高”转，而是可以讨论：

- 这个候选是 rediscovery 还是 extrapolation
- 它是不是 BN family 里合理的一支
- 它有没有足够的 BN-local 支持
- 它是不是更适合进入下一步结构工作流

### 3.3 项目开始具备“后续研究骨架”

以前这个项目更像一个终点停在 ranking 的 demo。

现在它已经形成了一个更完整的骨架：

- 数据标准化
- 分层 benchmark
- BN-focused 诊断
- 多层 candidate screening
- structure follow-up planning

这意味着后续如果要继续推进：

- BN-centered modeling
- 更强的 BN-aware 决策层
- 真实结构生成 / relaxation / stability validation

都有了一个比较清晰的落点，而不是还停在“先排个名再说”。

## 4. 你现在应该怎么对外描述这个项目

如果是给导师说，我建议用下面这个口径：

**这不是一个已经完成材料发现的系统，而是一个已经把评估、筛选、BN-focused 诊断和后续结构跟进逻辑拆清楚的研究型 PoC。**

你可以重点强调：

1. **项目从 generic demo 变成了 BN-themed PoC**
2. **项目从单一 benchmark 变成了分层评估**
3. **项目从一个 ranking 表，变成了多层候选输出 + 后续结构桥接**
4. **项目最难看的问题没有被藏起来，而是被显式诊断出来了**

最不建议说的话是：

- “这个系统已经能可靠发现新的 BN 材料”
- “现在 BN 泛化已经很好”
- “structure 部分已经做完”

这些说法现在都太大。

---

## 5. 我对当前阶段的判断

如果要直接回答“这个版本是不是值得重新发给导师看”，我的判断仍然是：

**值得。**

原因不是它已经做完了，而是它已经明显从：

- 一个泛化的工程 demo

推进到了：

- 一个边界更清楚、对批评更有回应能力、也更像博士研究入口的 BN-themed PoC。

更直白一点说：

- 它现在比旧版本更容易得到导师的“认可你们确实在认真修方法”的评价；
- 但它还不足以让一个非常严厉的导师直接认为“核心科学问题已经解决”。

所以当前最合适的定位是：

**这是一个明显进步、值得重新汇报、但仍需继续攻克 BN-centered generalization 与真实结构验证的版本。**

---

## 6. 当前最值得继续推进的方向

从当前代码和 artifact 看，接下来最值得继续做的不是再堆报告字段，而是继续强化下面两类问题至少一类：

1. **BN-centered modeling / decision layer**
   - 让 BN 不只是一个解释和诊断视角，而更深地进入模型或决策逻辑本身。

2. **真实结构后续验证**
   - 把当前的 structure seed / job plan / queue，推进到真实 structure workflow，而不是停留在 planning artifact。

如果这两条线继续推进，这个项目才更可能从“方法学上更像研究”继续走向“研究结论也开始站得住”。
