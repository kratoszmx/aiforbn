# 给见微的说明

## 一句话先说结论

这个项目目前已经做成了一个**能跑通、能演示、能继续往上叠研究工作的 AI for BN PoC**。

它现在最准确的定位不是“已经做出了一个很强的科研系统”，而是：

- 已经把主流程接通了
- 已经有可复现结果和可展示产物
- 已经能支持下一阶段做更严谨的方法升级
- 但目前模型还只是一个**偏弱的基线**，离“能让人很放心地信它的筛选结果”还有距离

如果把你当成一个略懂 AI、但不懂化学、同时更关心项目进度和风险的项目经理，那你最该关心的是：

1. 这个项目现在到底做到了哪一步
2. 它的技术路线是不是说得通
3. 当前结果值不值得继续投入
4. 下一步该优先补哪里

这份说明就按这个思路来讲。

---

## 1. 这个项目现在到底在做什么

这个项目当前在做的是：

**围绕 BN 相关二维材料，做一个 band gap 预测 + 候选排序的 AI 初筛系统。**

把它翻成更直白的话，就是：

- 先从公开数据库里拿一批材料数据
- 用一个轻量机器学习模型学习“材料组成”和“band gap”之间的大致关系
- 再生成一批 BN 风格候选材料
- 用模型给这些候选做预测和排序
- 最后把结果保存出来，用 notebook 和网页展示

所以当前它不是一个“自动发现新材料”的系统，而是一个：

**AI 辅助初筛 demo + 后续研究骨架**

---

## 2. 为什么选 BN，为什么选 band gap

### 为什么选 BN

BN 指的是硼氮化物（boron nitride）。

选它做第一版主题，主要有三个实际原因：

- **主题够聚焦**，不会一上来把问题做得太散
- **对外好讲**，即使不懂化学，也能理解“围绕一类材料做 AI 初筛”
- **后续容易扩展**，以后可以从 BN 扩到更大的 III-V 相关材料范围

换句话说，BN 适合做第一阶段的项目切口，不是因为它已经被当前系统深度解决了，而是因为它适合作为一个清晰的 PoC 范围。

### 为什么选 band gap

band gap 可以先粗略理解成材料一个很重要的性质指标，它和材料更像绝缘体、半导体还是更容易导电有关。

从项目管理角度，你只要记住它适合做第一版目标就行，因为：

- 它在公开材料数据里常见
- 是一个明确的数值标签
- 很适合先做回归基线
- 便于后续比较不同模型和特征

所以当前路线是合理的：

**先用一个公开数据里容易拿到、AI 任务定义清楚的目标，把整条流程跑起来。**

## 4. 整个工作流程，你应该怎么理解

你不需要记函数名，但需要知道这条链路是怎么串起来的。

### 第一步，拿数据

项目从公开二维材料数据源 `twod_matpd` 拉数据，并整理成统一表结构。

你可以把这一步理解成：

- 先把原材料样本池拿到手
- 再把格式整理成后面机器学习能直接用的样子

### 第二步，提取 BN 相关部分

项目会从全体材料里找出化学式中同时包含 B 和 N 的记录，形成一个 BN 主题切片。

这一步的作用更多是：

- 让项目主题聚焦
- 让展示更贴近“AI for BN”

但要注意，当前真正训练模型时，用的还是**整张数据表**，不是只用这 12 条 BN 样本。

### 第三步，把材料变成模型能读的输入

当前系统不是看晶体结构图，也不是做复杂量子化学计算。

它现在做的是：

- 从化学式出发
- 提取一些很基础的组成信息
- 例如元素个数、原子序数统计、是否含 B、是否含 N

也就是说，当前模型主要看到的是：

**“这个材料由哪些元素组成，它们大概属于什么元素级别”**

这是一个很轻量、很适合 PoC 的表示方式，但表达能力也有限。

### 第四步，训练一个基线模型

当前默认模型是：

- **HistGradientBoostingRegressor**

选择它不是因为它最先进，而是因为它对这种中小规模表格数据比较稳，训练快，适合先把基线搭起来。

所以当前模型路线可以总结为：

**公开材料数据 + 简单组成特征 + 轻量梯度提升回归基线**

### 第五步，生成候选材料并做排序

当前候选生成方式非常简单，本质上是按一组 III-V 风格元素做组合枚举。

这一步的优点是：

- 简单
- 可解释
- 能把 screening 这一步真的跑起来

缺点也很明显：

- 它更像演示型候选生成
- 不是严肃的化学候选发现方法
- 目前还没有稳定性、结构合理性、可合成性这些约束

### 第六步，输出结果给人看

结果会落到 `artifacts/`，然后可以通过 notebook 或 Streamlit 展示。

所以这个项目现在已经不是“脑中想法”，而是有实际交付物的。

## 5. 当前重要方法选择，应该怎么理解

这部分是你需要知道但不需要记细节的。

### 数据方法

- 数据源：JARVIS / 2DMatPedia
- 当前是公开数据驱动路线
- 好处是起步快、可复现、适合 PoC

### 化学/材料侧方法

当前并没有上更重的结构级方法，也没有做第一性原理验证。

实际做法更接近：

- 用化学式做轻量表示
- 用简单规则生成候选
- 先验证 AI 初筛链路能不能成立

所以你可以把当前化学方法理解成：

**偏轻、偏工程化、偏 PoC，不是深度材料机理建模。**

### AI 方法

当前是典型的表格数据监督学习基线：

- 输入：formula 提取出来的基础组成特征
- 输出：band gap
- 模型：梯度提升回归
- 评估：MAE / RMSE / R²

项目管理上，这个选择是合理的，因为它让项目先获得：

- 可跑通性
- 可复现性
- 可解释性
- 可替换性

以后要升级，也能很清楚地升级到更强特征和更强模型。

## 8. 当前项目最大的风险和局限

这是你作为项目经理最应该盯住的部分。

### 风险 1：模型还比较弱

当前 `R² = 0.159`，说明模型不是完全没学到东西，但也明显谈不上强。

这意味着：
- 当前结果可以支持“继续做”
- 但还不能支持“很有把握地信”

### 风险 2：评估可能偏乐观

这是当前最关键的方法风险。

当前数据里有不少重复 formula，但切分方式是随机按行切。

这会导致：
- train 和 test 里可能出现相同 formula
- 对于当前这种 formula-only 模型，容易出现信息泄漏风险
- 所以当前 test 分数可能偏乐观

这是后续必须优先修的点。

### 风险 3：BN 主题现在更多是“项目主题”，还不是“方法优势”

当前 BN 子集只有 12 条。

这说明：
- 目前项目确实围绕 BN 做了展示聚焦
- 但模型本身还不是一个真正“专门为 BN 建起来的强系统”

### 风险 4：候选生成偏演示化

当前候选不是经过严肃化学筛选得来的，而是规则枚举出来的。

所以目前更像：
- “先把候选排序系统做出来”
而不是：
- “已经建立了高可信候选发现系统”

### 风险 5：有些配置是预留，不是已完成功能

例如：
- `matminer`
- API
- LLM 相关开关

这些现在不能算成已实现能力，最多算成预留升级位。

## 10. 如果继续投入，最值得优先补什么

如果按投入产出比来排，我建议优先级如下：

### 第一优先级：先修评估

最该先做的是：

- 按 formula 分组切分
- 做更严谨的验证
- 重新看分数到底有多少真实泛化能力

因为如果评估方式有问题，后面再加复杂模型，意义也会被打折。

### 第二优先级：升级特征

比如：

- 真正接入 matminer
- 加更强的组成特征
- 如果条件允许，再加结构特征

这是最直接提升模型质量的路线。

### 第三优先级：把训练和推理解耦

要补上：

- 模型持久化
- 统一预测入口
- 让 notebook、网页、未来 API 共用同一推理逻辑

这样项目会从“脚本”更像“系统”。

### 第四优先级：把候选生成做得更可信

例如：

- 加入更合理的化学约束
- 不是只靠简单枚举
- 对 top candidates 再做二次过滤

### 第五优先级：把展示层升级

当前 Streamlit 只是结果查看器。

后续可以升级为：

- 输入 formula 即时预测
- 候选筛选页
- 指标诊断页
- BN 专题页
