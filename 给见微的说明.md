# 给见微的说明

## 一句话结论

这个项目现在最准确的定位是：

**一个已经稳定跑通、评估与筛选叙事更诚实、并且同时引入轻量 structure-aware evaluation、轻量化学可信度筛选层，以及 novelty / rediscovery 透明分层的 AI for BN 演示型 PoC。**

它现在已经不只是“一个能跑的 composition-only baseline”了，但也远没到“可以主张新材料发现能力”的阶段。

---

## 1. 现在到底做到了什么

当前系统做的是一件很具体的事：

**用公开二维材料数据做 band gap 预测，然后对一个很小的 BN / III-V toy 候选空间做 demo ranking。**

主流程现在已经稳定固定为：
- 读取或重建规范化数据；
- 按 `formula` 分组切 train / val / test；
- 同时构造三条特征路线；
- 在验证集上选最好的 `{feature_set} x {model_type}` 组合；
- 在测试集上做完整 benchmark；
- 再对没有结构的候选公式做公式级 ranking；
- 把关键 artifact、plot、summary 都落盘。

关键是：

**“评估最优组合”和“候选排序可用组合”现在已经被明确分开。**

这点很重要，因为现在最好的评估组合已经会用到结构摘要信息，但 toy 候选只有化学式，没有结构，所以排序时不能假装这些候选也有结构。

---

## 2. 这轮升级后，真正重要的变化

### 变化 1：系统已经有轻量 structure-aware 路径

现在不再只有两条 composition-only 路线，而是有三条特征路径：

- `basic_formula_composition`
  - 7 维手写基础控制组
- `matminer_composition`
  - 19 维 composition-only matminer 描述符
- `matminer_composition_plus_structure_summary`
  - 上面那 19 维 + 11 个从 cached raw JARVIS `atoms` / `lattice` 信息里提出来的结构摘要

这里的 structure-aware 还很轻，只是结构摘要，不是复杂图网络，也不是高阶材料物理建模。
但它已经足以让系统第一次出现一个方法学上更诚实的事实：

**评估最优路径和候选排序路径不一定相同。**

### 变化 2：系统现在会诚实地区分 evaluation 和 screening

这次验证后，两个组合分别是：

- overall evaluation 最优：
  - **`matminer_composition_plus_structure_summary + hist_gradient_boosting`**
- formula-only screening 最优：
  - **`matminer_composition + hist_gradient_boosting`**

也就是说：
- 测试集评估会承认 structure-aware 组合整体更好；
- 但候选排序只能用候选兼容的 formula-only 组合；
- 系统不会再把“最好的测试集组合”直接拿去给没有结构的候选打分。

这就是这轮 wave-3 最核心的升级。

### 变化 3：结构路径确实带来了提升，但只是小幅、不是飞跃

当前测试集结果里：

- structure-aware 最优组合：`MAE = 0.5568`
- 最好的 formula-only 组合：`MAE = 0.5822`

也就是：

- 绝对改善约 `0.0254`
- 相对改善约 `4.4%`

验证集上也是同样方向：

- structure-aware HGB：`MAE = 0.5724`
- composition-only HGB：`MAE = 0.6006`

所以现在最诚实的说法不是“结构特征带来了巨大提升”，而是：

**轻量 structure-aware 路径带来了小幅但一致的评估改善，足以改变最优 evaluation combo，但还没有形成压倒性优势。**

### 变化 4：候选排序仍是 demo，但现在多了一层基础化学可信度约束

当前候选排序仍然是一个 toy 空间：

- Group 13：`B, Al, Ga, In, Tl`
- Group 15：`N, P, As, Sb, Bi`
- 两两组合共 25 个公式

但现在不再是“25 个公式直接一起打分完就结束”。

系统会先用 `pymatgen` 的 oxidation-state guesses 给每个候选写出一层**公式级 chemical plausibility annotation**，例如：
- `chemical_plausibility_pass`
- `chemical_plausibility_guess_count`
- `chemical_plausibility_primary_oxidation_state_guess`
- `chemical_plausibility_note`

然后在输出 artifact 时：
- 保留完整 25 个 source-space 候选；
- 明确写出哪些公式进入最终 top-k；
- 对没有通过基础 charge-balance 检查的公式做显式降级，而不是混在正常候选里假装同等可信。

当前这层轻量 screening 的直接结果是：
- **21 / 25** 个公式通过；
- **4 / 25** 个公式未通过；
- 当前未通过的是：`AlBi`, `GaBi`, `InBi`, `TlBi`

输出时现在会写：
- `predicted_band_gap`
- `ensemble_predicted_band_gap_mean`
- `ensemble_predicted_band_gap_std`
- `ranking_score`
- `chemical_plausibility_pass`
- `chemical_plausibility_primary_oxidation_state_guess`
- `screening_selected_for_top_k`
- `screening_selection_decision`
- `seen_in_dataset`
- `seen_in_train_plus_val`
- `candidate_novelty_bucket`
- `candidate_is_formula_level_extrapolation`
- `novelty_rank_within_bucket`
- `novel_formula_rank`

这意味着现在的排名不再只是一个单点分数，而是会同时暴露：
- 基础化学式可信度；
- 小模型池分歧；
- 候选是否其实早就在数据里；
- 它到底是 train+val rediscovery、held-out-known formula，还是 formula-level extrapolation。

所以它现在更像：

**sanity-check ranking / pipeline demo**

而不是：

**新材料发现结论。**

### 变化 5：系统现在会把“重放已知公式”和“真正未见公式”分开讲

这轮新增的 novelty / rediscovery layer 直接回答了一个老师会很在意的问题：

**当前 top-k 到底是在做 rediscovery，还是在做真正的外推？**

最新 artifact 里，25 个 toy candidates 会被显式分成三类：
- `train_plus_val_rediscovery`：**23** 个
- `held_out_known_formula`：**1** 个，当前是 `AlP`
- `formula_level_extrapolation`：**1** 个，当前是 `BBi`

而标准 top-k 20 的构成是：
- rediscovery：**19** 个
- held-out-known：**1** 个
- formula-level extrapolation：**0** 个

也就是说，当前系统现在已经会诚实地承认：
- 这轮 top-k 基本还是已知公式重放；
- 当前唯一真正 dataset 未见的 toy formula 是 `BBi`；
- 它虽然通过了轻量 plausibility 检查，但最终 `ranking_rank = 21`，没有进入 top-k。

这不会 magically 把系统变成 discovery engine，但它确实让 artifact 更像博士级项目该有的样子：

**不再把 rediscovery、held-out-known、以及 formula-level extrapolation 混成同一种“候选”。**

## 3. 最新一次已验证结果

这些不是计划，是刚刚重新跑过的结果。

### 数据规模

- 总数据：**6351** 条
- 唯一 `formula`：**4381** 个
- BN 主题切片：**12** 条
- BN 唯一 `formula`：**10** 个

### 默认 split

- train：**5101** rows / **3505** formulas
- val：**618** rows / **438** formulas
- test：**632** rows / **438** formulas

并且：

- `train/val/test` 的 `formula overlap = 0`

### 联合选择结果

验证集当前比较的是 6 个组合：

- `basic_formula_composition + hist_gradient_boosting`
- `basic_formula_composition + linear_regression`
- `matminer_composition + hist_gradient_boosting`
- `matminer_composition + linear_regression`
- `matminer_composition_plus_structure_summary + hist_gradient_boosting`
- `matminer_composition_plus_structure_summary + linear_regression`

其中最优的是：

- **overall evaluation：`matminer_composition_plus_structure_summary + hist_gradient_boosting`**
- **formula-only screening：`matminer_composition + hist_gradient_boosting`**

### 当前测试集指标

被选中的 overall evaluation 组合测试结果为：

- **MAE = 0.5568**
- **RMSE = 0.8218**
- **R² = 0.6302**

### benchmark 对照

- `matminer_composition_plus_structure_summary + hist_gradient_boosting`: **MAE = 0.5568**
- `matminer_composition + hist_gradient_boosting`: **MAE = 0.5822**
- `basic_formula_composition + hist_gradient_boosting`: **MAE = 0.9219**
- `matminer_composition_plus_structure_summary + linear_regression`: **MAE = 0.8198**
- `matminer_composition + linear_regression`: **MAE = 0.8555**
- `basic_formula_composition + linear_regression`: **MAE = 1.0056**
- `dummy_mean`: **MAE = 1.1118**

这说明三件事：

1. 结构摘要路径比最好的 composition-only 路径更好，但提升不大。
2. composition-only 的 richer path 明显优于最弱的手写控制组。
3. 当前系统已经显著优于 mean predictor，但这仍然只是一个 PoC 级 baseline。

### 当前 ranking 的前列候选

现在排在前面的还是：

- `BN`
- `AlN`
- `GaN`
- `AlP`
- `InN`

而排在 artifact 末尾、并被明确标成 `failed_chemical_plausibility` 的是：

- `InBi`
- `TlBi`
- `GaBi`
- `AlBi`

与此同时，当前 novelty 分层也给出了更诚实的解释：
- `AlP` 不是新公式，而是 **held-out-known formula**；
- 当前唯一 **formula-level extrapolation** 是 `BBi`；
- `BBi` 通过了轻量 plausibility，但 `ranking_rank = 21`，没有进 top-k。

这说明现在的 candidate-space 表达至少比之前更诚实：
- 上游 source space 仍然透明；
- 下游不会再把明显连基础 charge-balance 都过不了的公式与其他候选放在同一可信度层级里；
- 也不会再把 rediscovery 和真正 dataset 未见公式混成同一类“候选发现”。

## 4. 你应该怎么理解它现在的价值

### 它已经解决了什么

- 默认评估协议更可信了；
- 现在真的有 baseline comparison；
- 结构信息已经以轻量方式接进来了；
- evaluation 和 screening 的逻辑边界已经被明确写进 artifact；
- 候选排序不会再假装没有结构的候选也能直接享受 structure-aware 最优模型。

### 它最像什么

它现在最像：

**一个研究骨架、展示骨架、以及方法学自我约束都已经成形的材料 AI PoC。**

### 它还不像什么

它还不像：

- 可直接主张结构级筛选能力的系统；
- 可直接指导实验的高可信候选发现工具；
- 真正面向 BN 专题建模的成熟研究系统。

---

## 5. 现在最重要的限制

这里直接说，不包装。

### 限制 1：structure-aware 还是很轻量

现在的 structure-aware 路径只用了从 cached raw JARVIS `atoms` 数据里提出来的结构摘要。

它还没有真正做到：
- 晶体图表示；
- 更强的结构编码；
- 稳定性 / 形成能 / 声子稳定性联合约束。

### 限制 2：candidate ranking 仍然只能是 formula-only

因为当前 toy candidates 没有结构，所以候选排序只能退回到 `matminer_composition` 这条公式级路径。

这恰恰说明系统现在更诚实了，但也说明：

**当前 screening 能力本身并没有升级成 structure-aware screening。**

### 限制 3：BN slice 仍然太小

BN 主题切片只有 12 条记录，10 个唯一公式。

所以当前训练主体仍然是全体二维材料，而不是 BN-only 学习。

### 限制 4：候选空间太玩具，而且真正未见公式太少

25 个 Group 13/15 组合没有结构、稳定性、合成可行性约束。

更关键的是，在当前这 25 个公式里：
- 24 个本来就在 dataset 里出现过；
- 23 个甚至已经在 `train+val` 里出现过；
- 真正 dataset 未见的只有 `BBi` 一个。

因此当前结果最多只能说明：
- pipeline 能跑；
- ranking 行为大体合理；
- 系统现在能诚实地区分 rediscovery / held-out-known / formula-level extrapolation；
- 但这仍然不是 discovery evidence。

## 7. 下一步最值得做什么

如果继续往更像研究的方向推，优先级应该是：

1. 继续把 structure-aware 路径做强，而不是停在轻量摘要层。
2. 把 toy candidate space 变得更像真正 screening 空间，而不是大多数都只是已知公式重放。
3. 给候选空间加入更可信的结构 / 稳定性约束。
4. 把当前 disagreement heuristic 升级成更可信的不确定性处理。
5. 进一步解耦 train / inference，减少 demo 脚本意味。
