# 给见微的说明

## 一句话结论

这个项目现在最准确的定位是：

**一个已经把主流程跑通、方法学比之前更诚实、但仍然只是 PoC 的 AI for BN 演示系统。**

它已经不只是“有个想法”，而是有真实数据、真实评估、真实 artifact 和可展示页面；但它也还远远不是“可以拿去做强科研结论”的系统。

---

## 1. 这个项目现在到底在做什么

当前项目做的是一件很具体的事：

**用公开二维材料数据做 band gap 预测，然后对一个很小的 BN/III-V 风格 toy 候选空间做 demo 排序。**

把它翻成更直白的话，就是：

- 从公开数据库拿材料数据；
- 把数据整理成统一表格；
- 用化学式构造很基础的组成特征；
- 训练一个轻量回归模型；
- 在严格一些的测试协议下评估它；
- 再对一组演示型候选做打分；
- 把结果保存成可检查的文件，并给 notebook / Streamlit 展示。

所以它现在不是“新材料发现系统”，更不是“已经很强的科研模型”，而是：

**一个可运行的 AI 初筛 PoC。**

---

## 2. 这次升级后，最重要的变化是什么

这次不是单纯修代码，而是补最关键的方法学短板。

### 变化 1：默认评估 split 已经改严了

之前最危险的问题是：

- 数据里有很多重复 `formula`；
- 如果随机按“行”切 train/test；
- 那么同一个 `formula` 很可能同时出现在训练集和测试集；
- 对当前这种 formula-only 模型，这会带来明显泄漏风险。

现在默认切分已经改成：

**按 `formula` 分组切分。**

也就是说，同一个 `formula` 只会出现在 train、val、test 其中一个集合里，不会跨集合重复。

这是当前这轮升级里最重要的一项。

### 变化 2：加了最基础的 benchmark

现在不再只报一个模型分数。

系统会同时输出：

- 被选中的模型；
- 另一个小候选模型；
- `dummy_mean` 这样的 naive baseline。

这样至少可以回答一个最基本的问题：

**当前模型到底有没有比“瞎猜平均值”更强。**

### 变化 3：验证集现在真的在干活

现在不是固定死一个模型直接跑到底。

系统会先在验证集上，在一个很小的候选模型集合里做选择：

- `linear_regression`
- `hist_gradient_boosting`

选出验证集表现更好的那个，再用 `train + val` 重新训练，并在 test 上报告结果。

这一步很克制，没有过度调参，但比“完全不用验证集”更合理。

### 变化 4：matminer 相关说法已经纠正

现在配置里已经明确写成：

- `feature_set = basic_formula_composition`
- `matminer_enabled = False`

所以当前系统的真实状态很清楚：

**它现在只用了手写的基础组成特征，并没有接上 matminer。**

### 变化 5：候选排序不再硬叫 screening discovery

现在候选空间被明确命名为：

`toy_iii_v_demo_grid`

输出文件也改成了：

`demo_candidate_ranking.csv`

文件里还会写上：

- 这是 toy/demo 候选空间；
- 它只是 Group 13 / Group 15 的简单枚举；
- 没有稳定性、结构合理性、可合成性约束。

这能避免把演示型结果说成严肃候选发现。

## 3. 现在的系统规模和最新结果

最近一次已经实际跑通并验证的结果如下。

### 数据规模

- 总数据：**6351** 条
- 唯一 `formula`：**4381** 个
- BN 主题切片：**12** 条
- BN 唯一 `formula`：**10** 个

### 默认 split

默认是 grouped-by-formula：

- train：**5101** rows / **3505** formulas
- val：**618** rows / **438** formulas
- test：**632** rows / **438** formulas

最关键的是：

- train / val / test 的 `formula overlap = 0`

### 选模结果

系统先在验证集上比较：

- `linear_regression`
- `hist_gradient_boosting`

这次实际选出来的是：

- **`hist_gradient_boosting`**

### 测试集结果

被选中模型的 test 指标是：

- **MAE = 0.9114**
- **RMSE = 1.1813**
- **R² = 0.2358**

### benchmark 对照

同一轮 test benchmark 里：

- `hist_gradient_boosting`: **MAE = 0.9114**
- `linear_regression`: **MAE = 1.0056**
- `dummy_mean`: **MAE = 1.1118**

这说明什么？

- 当前模型确实比“平均值瞎猜”更好；
- 但提升幅度并没有大到足以支持强科研结论；
- 它更像“有一定信号的弱基线”，不是强模型。

## 4. 你应该怎么理解这个项目现在的价值

### 它已经解决了什么

- 不再只是概念；
- 不再只是零散脚本；
- 不是只有一个不可信的 test 分数；
- 而是已经有一条比较完整、比较诚实的端到端工作流。

### 它还没有解决什么

- 还没有结构级特征；
- 还没有 matminer 特征；
- 还没有不确定性估计；
- 候选空间仍然是玩具级 demo；
- BN 本身还主要是“项目主题”和“展示切口”，还不是方法上的专门优势。

所以这项目现在最真实的价值是：

**研究骨架已经搭好了，而且最明显的方法学漏洞已经补了一轮。**

但真正有说服力的研究工作，还在后面。

## 5. 现在主要会产出什么文件

跑完 `python main.py` 之后，当前最重要的结果都在 `artifacts/` 里。

你最该看的几个文件是：

- `metrics.json`
  - 当前被选模型的测试集指标和元数据。
- `benchmark_results.csv`
  - 候选模型和 dummy baseline 的测试集对比。
- `experiment_summary.json`
  - 数据规模、split 协议、选模结果、候选空间说明。
- `demo_candidate_ranking.csv`
  - 当前 toy/demo 候选空间的排序结果。
- `bn_slice.csv`
  - BN 主题切片。
- `parity_plot.png`
  - 最基础的预测对比图。

如果你只想判断“项目现在是不是比之前更靠谱”，重点看前三个文件就够了。

## 6. 现在最清楚的限制是什么

这里直接说结论，不包装。

### 限制 1：现在仍然只是 composition-only baseline

模型输入目前只来自化学式推出来的基础组成统计。

这意味着它还没有真正看到：

- 结构信息；
- 更丰富的材料描述符；
- 更接近材料机理的信息。

### 限制 2：BN 主题样本本身非常小

BN 切片只有 12 条记录，10 个唯一 formula。

所以当前训练主体实际上还是全体二维材料数据，而不是 BN-only 训练。

### 限制 3：候选空间还是 toy/demo

当前候选只是一组很小的元素组合枚举。

它能证明的是：

- 排序链路能跑；
- artifact 能落地；
- 模型输出至少没有完全胡来。

它不能证明的是：

- 系统已经有可信的材料发现能力；
- top candidates 可以直接指导实验。

### 限制 4：还没有不确定性和真正推理解耦

现在结果更像一次离线跑批。

如果后面要继续往研究工具或服务化走，还需要：

- inference 接口；
- uncertainty-aware ranking；
- 更明确的 train / inference 解耦。

## 7. 现在值不值得继续投

值。

但理由不是“它已经很强”，而是：

- 它已经有真实工作流；
- 最危险的方法学问题已经补了一轮；
- 下一步该做什么现在很清楚；
- 后续每一步升级都有明确落点。

如果继续推进，我建议优先级仍然是：

1. 更强但仍然诚实的特征表示。
2. 更可信的候选空间约束。
3. 不确定性和推理解耦。

**一个已经证明了高可信候选发现能力的科研系统。**
