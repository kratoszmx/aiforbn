# 给见微的说明

## 一句话结论

这个项目现在最准确的定位是：

**一个已经稳定跑通、评估逻辑更诚实、并且把候选空间推进到 BN-anchored formula-family demo space，同时把 BN-local support、BN analog evidence、BN analog validation proxy，以及 grouped-by-formula robustness 检验都接进主流程的 AI for BN 演示型 PoC。**

它已经不只是“能跑的 composition-only baseline”，但也还远没到“可以主张新材料发现能力”的阶段。

---

## 1. 现在到底做到了什么

当前系统做的是一件很具体的事：

**用公开二维材料数据做 band gap 预测，然后对一个小型、BN 主题明确的公式候选空间做 demo ranking。**

主流程现在固定为：
- 读取或重建规范化数据；
- 按 `formula` 分组切 train / val / test；
- 同时构造三条特征路线；
- 在验证集上联合选择 `{feature_set} x {model_type}`；
- 在测试集上做完整 benchmark；
- 再对没有结构的候选公式做 formula-only ranking；
- 把关键 artifact、plot、summary 都落盘。

关键变化有六个：

1. **“评估最优组合”和“候选排序可用组合”已经被明确分开。**
2. **候选空间不再默认是旧的 Group-13/Group-15 笛卡尔 toy grid，而是 BN-anchored family grid。**
3. **BN 不再只是报告主题，screening 里现在多了一层 BN-local support 参考。**
4. **候选解释不再只看距离和分数，还会调出邻近 BN analog 的真实观测属性。**
5. **这些 analog 证据不再只是原始数字，而会被压缩成一个更直观的 analog-validation label，并进一步进入轻量 ranking penalty。**
6. **候选排序现在还会额外看 grouped-by-formula fold-to-fold spread，不再只依赖一次 `train+val` 拟合。**

这意味着系统现在既更像一个材料方向的 PoC，也更诚实。

---

## 2. 这轮最重要的升级

### 变化 1：evaluation 仍然是三条特征路线联合比较

当前仍保留三条特征路径：

- `basic_formula_composition`
  - 7 维手写基础控制组
- `matminer_composition`
  - 19 维 composition-only 描述符
- `matminer_composition_plus_structure_summary`
  - 19 维 composition 特征 + 11 个从 cached raw JARVIS `atoms` / `lattice` 派生的结构摘要

所以现在的核心事实仍然成立：

- **overall evaluation 最优组合可以是 structure-aware**；
- **但 formula-only 候选排序不能假装自己也有结构。**

当前验证后的两个组合分别是：

- overall evaluation 最优：
  - **`matminer_composition_plus_structure_summary + hist_gradient_boosting`**
- formula-only screening 最优：
  - **`matminer_composition + hist_gradient_boosting`**

测试集结果仍然是：

- structure-aware 最优：`MAE = 0.5568`
- 最佳 formula-only：`MAE = 0.5822`

也就是轻量结构摘要路径仍然带来**小幅但一致**的改善，但不是飞跃。

而且我这一步又补了一层以前还不够强的地方：

- 不再只看这一个 holdout test split；
- 现在还会额外跑 **grouped-by-formula 的 5-fold robustness benchmark**。

最新 grouped robustness 结果里：

- `matminer_composition_plus_structure_summary + hist_gradient_boosting`
  - `MAE_mean = 0.6050 ± 0.0176`
- `matminer_composition + hist_gradient_boosting`
  - `MAE_mean = 0.6321 ± 0.0192`
- `dummy_mean`
  - `MAE_mean = 1.1527 ± 0.0233`

这说明“structure-aware evaluation 略优于最佳 formula-only fallback”这件事，现在**不只是在一次 split 上成立**，在 grouped-by-formula 的重复切分里也基本站得住。

而且 grouped robustness 这次不只停留在 evaluation 报表层。

我又把它继续接回 candidate ranking 本身了：

- 用 selected formula-only screening combo 在 **train+val 的 grouped folds** 上重复训练；
- 对每个 candidate 计算 fold-to-fold prediction spread；
- 再把这个 spread 作为一个很轻的 robustness penalty。

当前 live artifact 里：

- `grouped_robustness_penalty_weight = 0.15`
- `grouped_robustness_prediction_fold_count = 5`
- `grouped_robustness_penalized_rows = 25`
- `grouped_robustness_prediction_std_mean = 0.3660`

这层的意思不是“更大的 std 就一定更差”，而是：

**如果一个候选在 grouped-by-formula 的不同训练折里摇摆更大，那它的排序就不该完全等同于一个更稳的候选。**

### 变化 2：candidate space 已经从旧 toy grid 换成 BN-anchored family demo space

现在默认候选空间是 **25 个 BN-containing formulas**，而不是旧的 III-V 二元组合网格。

这个新空间由三类母题锚定：

- `BN` 二元 anchor
- `BCN / BC2N` 一类文献中常见的 BCN family 母题
- `Si2BN` 一类在当前数据集中真实出现过的 BN ternary motif

因此它比旧 toy grid 更符合老师那句“BN 要更 central”的要求。

当前 25 个候选按 family 分布为：

- `bn_binary_anchor`: 1
- `group14_bn_111_family`: 4
- `group14_bn_211_family`: 4
- `group14_bn_121_family`: 4
- `group13_bn_111_family`: 4
- `group13_bn_211_family`: 4
- `group13_bn_121_family`: 4

输出 artifact 里现在也会显式写出：

- `candidate_generation_strategy`
- `candidate_space_name`
- `candidate_family`
- `candidate_template`
- `candidate_family_note`

也就是说，现在不只是“给 25 个公式打分”，而是能看出**这些公式到底来自哪一种 BN family 设计假设**。

### 变化 3：chemical plausibility 层还在，但对象已经换成 BN family 候选

轻量 chemical plausibility screen 仍然使用：

- `pymatgen` oxidation-state guesses

它不是稳定性判断，也不是实验可实现性判断，只是一个**公式级 credibility boundary**。

在新的 BN-anchored candidate space 上，结果是：

- **23 / 25** 个候选通过
- **2 / 25** 个候选未通过
- 当前失败的是：
  - `AlBN`
  - `TlBN`

这比旧空间那种 `AlBi / GaBi / InBi / TlBi` 的失败模式更贴近当前 BN 主题。

### 变化 4：novelty layering 现在终于真的开始有内容了

这是这轮最重要的实质进步之一。

旧 toy grid 最大的问题是：几乎全是已知公式重放。

而在新的 BN-anchored candidate space 里，最新 artifact 显示：

- `train_plus_val_rediscovery`：**3** 个
  - `BN`
  - `BC2N`
  - `Si2BN`
- `held_out_known_formula`：**0** 个
- `formula_level_extrapolation`：**22** 个

标准 top-k 20 的构成为：

- `train_plus_val_rediscovery`：**3** 个
- `held_out_known_formula`：**0** 个
- `formula_level_extrapolation`：**17** 个

这和上一轮完全不一样。

现在已经不能再说“top-k 基本还是已知公式重放”，更准确的说法是：

**当前 top-k 已经以 formula-level extrapolation 为主，但这仍然只是公式级外推，不是结构级发现。**

### 变化 5：现在不只看全局 support，还会看 BN-local support、BN analog evidence 和 analog-validation penalty

上一轮虽然已经有了 domain-support，但它主要还是在回答：

- 候选离整个 train+val 数据分布远不远

这一轮先多接了一层 **BN-local support**，它回答的是：

- 候选离已知 BN slice 远不远
- 它最像哪一个已知 BN 公式

最新 artifact 里：

- `domain_support_penalized_rows = 12`
- `bn_support_reference_formula_count = 10`
- `bn_support_penalized_rows = 15`
- `bn_support_low_support_rows = 15`

例如当前 top-ranked 候选里，artifact 已经会直接给出 BN-local 最近锚点：

- `BCN2 -> B2(CN2)3`
- `BCN -> BC2N`
- `AlBN2 -> Si2BN`

但这轮更重要的是又多了一层 **BN analog evidence**。

它不直接假装预测 unseen candidate 的稳定性，而是去看：

- 它邻近的 BN analog 在真实数据里 band gap 大概是多少
- `energy_per_atom` 大概是多少
- `exfoliation_energy_per_atom` 大概是多少
- 磁性量级大概是多少

最新 summary 里：

- `bn_analog_evidence_enabled = True`
- `bn_analog_reference_formula_count = 10`
- `bn_analog_reference_exfoliation_energy_median = 0.07173938218750031`
- `bn_analog_reference_energy_per_atom_median = -7.929051850789474`
- `bn_analog_reference_abs_total_magnetization_median = 0.0`
- `bn_analog_exfoliation_available_rows = 25`
- `bn_analog_lower_or_equal_reference_rows = 2`
- `bn_analog_higher_reference_rows = 23`

这层最重要的意义不是“又加了一个分数”，而是：

**现在不只知道它离 BN 近不近，还能看到它所依附的 BN analog 在真实数据里的属性画像。**

而且这一步我又往前推了一层，新增了 **BN analog validation proxy**，把 analog 的 `exfoliation / energy / magnetization` 是否更接近 BN reference 区间，压缩成一个更可读的判断：

- `reference_like_on_available_metrics`
- `mixed_reference_alignment`
- `reference_divergent_on_available_metrics`

当前真实计数是：

- `reference_like = 2`
- `mixed = 2`
- `reference_divergent = 21`
- `bn_analog_validation_penalized_rows = 23`
- `bn_analog_validation_penalty_weight = 0.12`

例如：

- `BC2N` 属于 `reference_like`
- `BCN2 / BCN` 属于 `mixed`
- `AlBN2 / SiBN2 / SiBN` 这些高排位候选目前属于 `reference_divergent`

所以现在不只是知道“它分高不高”，还知道：

- 它靠近的是哪类已知 BN 公式
- 它背后的 BN analog 在观测属性上是什么样
- 它在一个更克制的 analog-validation 标签里属于哪一类
- 这个标签会不会进一步给它一个轻量 penalty
- 它是不是在 BN 主题内部也显得过于离谱

## 4. 现在这个系统最像什么

它现在最像：

**一个研究骨架、展示骨架、方法学约束骨架都已经比较完整，而且 candidate-space 叙事终于开始围绕 BN 本身组织起来的材料 AI PoC。**

它不再只是：

- 一个 generic 2D-material band-gap baseline
- 再额外硬挂一个 BN 标签

而是已经开始把：

- BN 主题
- 候选 family 设计
- ranking artifact 解释
- novelty layering
- BN-local support 参考
- BN analog evidence
- BN analog validation label
- BN analog validation penalty
- grouped candidate robustness penalty

放在同一条叙事线上。

## 5. 现在最重要的限制

这里直接说，不包装。

### 限制 1：candidate ranking 仍然是 formula-only

虽然候选空间更像 BN 项目了，但候选依然没有结构。

所以当前 screening 依然只能用：

- `matminer_composition`

它还不是 structure-aware candidate screening。

### 限制 2：formula-level extrapolation 不等于 discovery

现在 `22 / 25` 候选是 dataset 未见公式，这当然比上一轮强很多。

但这些“未见”只表示：

- **公式在当前 dataset 没出现过**

并不表示：

- 结构稳定
- 热力学可实现
- 可合成
- 真正值得实验优先验证

### 限制 3：structure-aware 路径还是很轻量

当前 structure-aware 仍然只是：

- composition features
- 加上一小组由 raw `atoms` 派生的结构摘要

还不是：

- crystal graph
- 更强结构编码
- 稳定性 / 形成能 / 声子稳定性联合建模

### 限制 4：BN slice 仍然太小

BN 主题切片仍然只有：

- 12 条记录
- 10 个唯一公式

所以当前训练主体仍然是整个二维材料表，而不是 BN-only 学习。

### 限制 5：uncertainty 仍然只是 heuristic

现在的 disagreement、domain-support、BN-local support、BN analog evidence、BN analog validation label 都比之前更有用了，但它们仍然只是：

- 小模型池分歧
- 全局特征空间近邻支持度
- BN slice 局部近邻支持度
- 基于已知 BN reference formulas 的 analog retrieval 证据
- 从 analog 证据压缩出来的轻量 validation proxy / ranking penalty

不是 calibrated uncertainty。

## 7. 下一步最值得做什么

如果继续往更像研究的方向推，优先级我会这样排：

1. **给 BN family candidates 加更可信的结构 / 稳定性约束。**
2. **把 formula-level extrapolation 往 structure-level candidate validation 推。**
3. **继续强化 structure-aware 表示，不停留在结构摘要层。**
4. **把当前 heuristic uncertainty 升级成更可辩护的不确定性估计。**
5. **进一步解耦 train / inference，减少 demo 脚本意味。**
