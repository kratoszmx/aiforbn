# 给见微的说明

## 一句话结论

这个项目现在最准确的定位是：

**一个已经从“能跑的 band-gap demo”升级成“方法学更诚实、BN 主题更明确、并且开始把候选排序桥接到后续结构工作流，甚至已经 materialize 出第一批 unrelaxed prototype”的研究型 PoC。**

它已经明显比早期版本更像一个可以拿去和导师讨论的博士项目雏形，但还没有到“可以直接主张发现新材料”的阶段。

## 0. 最新补充：modern-model wave 已经推进到 ensemble 稳定版

这次不是只做文献调研，我已经把一条真实的 modern neural 路线接进主流程，而且又继续推进了一步，不再停留在单个 seed 的 `torch_mlp`。

新增内容：
- 新 feature set：`fractional_composition_vector`
  - 118 维原始元素分数向量
  - 仍然是 formula-only、candidate-compatible
- 新模型：
  - `torch_mlp`
  - `torch_mlp_ensemble`
- 其中 `torch_mlp_ensemble` 是多 seed 平均预测版本
  - 目的不是“堆模型”，而是减少幸运 seed 影响
  - 并把 member-level 预测接进当前 uncertainty / ranking 体系

本轮真实验证结果：
- `pytest -q`：`28 passed, 5 warnings`
- `main.py`：运行成功
- 本机 PyTorch / Metal 可用：`torch 2.11.0`，`mps = True`

最重要的结果不是它把 overall benchmark 改写了，而是：
- overall best 仍然是 `matminer_composition_plus_structure_summary + hist_gradient_boosting`
- 但在 **严格 BN candidate-compatible 诊断** 上，当前最强 neural baseline 已经进一步升级为：
  - `matminer_composition + torch_mlp_ensemble`
  - BN formula holdout `MAE = 1.1257`，优于单个 `torch_mlp = 1.1942`，也优于 `dummy_mean = 1.3257`
  - BN family holdout `MAE = 1.1382`，优于单个 `torch_mlp = 1.2032`，也优于 `dummy_mean = 1.3255`
  - BN vs non-BN grouped error ratio `≈ 1.8109`

这代表什么：
- 旧结论“模型太老”现在已经更站不住了，因为项目里不再只有传统浅层控制组
- 而且第一条 candidate-compatible modern neural 路线已经从“单个 seed 可行”推进到“multi-seed ensemble 版本更强”
- 但 BN-centered generalization **仍未被解决**，只是当前已经出现了一条更值得继续推进的 candidate-compatible 现代模型路线
- 所以现在最诚实的话术应升级为：
  - **这个项目已经进入 modern-model 阶段，并且已经有一条在严格 BN 诊断下站得住、而且不只是幸运 seed 的 neural baseline**
  - 但还不能说 BN 问题已经解决

另外，`BN-centered alternative ranking` 现在已经自动切换为：
- `matminer_composition + torch_mlp_ensemble`

这说明新模型不只停留在 benchmark CSV 里，而是已经进入项目真正的 BN-centered control view。

## 1. 现在这个项目到底做到了什么

如果不看代码、只看你最需要知道的结论，可以概括成四件事：

1. **评估更可信了**
   - 主评估继续使用按 `formula` 分组切分，避免同一公式泄漏到 train/val/test。
   - 现在还额外做了 `grouped-by-formula` 的 5-fold robustness benchmark，不再只依赖一次 holdout 结果。
   - 又专门加了 **BN-slice leave-one-BN-formula-out** 诊断，专门检查 BN 方向到底有没有真的泛化。

2. **BN 终于更“在项目中心”了**
   - 候选空间已经从旧的 generic toy grid 改成 **BN-anchored formula-family grid**。
   - 当前候选生成围绕 `BN / BCN / BC2N / Si2BN` 一类 BN 相关母题组织，不再像以前那样只是给 generic 候选贴 BN 标签。

3. **候选筛选不再只是一个 top-k 分数表**
   - 现在会同时输出：
     - raw ranking
     - proposal shortlist
     - extrapolation shortlist
     - BN-centered alternative ranking
     - ranking uncertainty / abstention / decision-policy table
   - 同时加入 `BN-local support`、`BN analog evidence`、`BN analog validation penalty`、`grouped robustness penalty`、`top-k stability` 等解释层。

4. **项目已经开始回答“排完名之后下一步做什么”**
   - 现在不仅有候选排名，还有 structure seed、handoff JSON、reference records、job plan、first-pass queue、prototype-grounded follow-up shortlist，以及第一批 first-pass execution 的 JSON / CSV / `.cif` 原型输出。
   - 也就是说，系统已经不只是在说“排完名之后该怎么做”，而是已经把一小批高优先候选真正变成了可交给后续 relaxation workflow 的 unrelaxed prototype。

## 2. 当前最重要的结果应该怎么理解

### 2.1 一般二维材料 band-gap 预测这件事，当前系统已经做得像样了

当前主 benchmark 下：

- 最优 overall evaluation 组合：
  - `matminer_composition_plus_structure_summary + hist_gradient_boosting`
- test 指标：
  - `MAE = 0.5568`
  - `RMSE = 0.8218`
  - `R² = 0.6302`

最佳 formula-only 路径为：

- `matminer_composition + hist_gradient_boosting`
- test `MAE = 0.5822`

这意味着：

- 在一般二维材料 band-gap 预测上，当前轻量 structure-aware 路径相对最佳 composition-only 路径，确实有**小幅但一致**的提升。
- 这件事不仅出现在单次 test split 上，在 grouped-by-formula 的 5-fold robustness benchmark 里也成立。

所以如果只问“这个项目是不是已经比一个普通 baseline demo 强很多”，答案是**是的**。

### 2.2 但 BN-centered generalization 仍然是主要短板

这也是现在最应该诚实说明的地方。

当前新增的 BN-slice benchmark 会把每个 BN 公式轮流留出，专门看 BN 方向的外推。结果显示：

- selected model：`MAE = 1.6397`
- screening model：`MAE = 1.6383`
- global dummy baseline：`MAE = 1.3257`
- 当前 BN-slice 上最强的 **candidate-compatible** 组合：
  - `matminer_composition + torch_mlp_ensemble`
  - `MAE = 1.1257`
- 当前 BN-slice 上最强的 overall 组合：
  - `matminer_composition_plus_structure_summary + torch_mlp`
  - `MAE = 1.0693`

这代表：

- 当前整体最优模型和当前默认筛选模型，在 BN-slice 上都**没有打赢** dummy baseline；
- 但这次 modern-model wave 之后，已经首次出现了一个在严格 BN 公式留出诊断下**明确打赢 dummy** 的 candidate-compatible neural baseline。

所以现在最准确的话术不是：

- “我们已经把 BN 做出来了”

而是：

- “我们已经把 BN 方向的问题单独拆出来认真测了，而且测出来它目前仍然是主要短板。”

这其实很重要，因为它让项目从“看起来有很多东西”变成“知道自己哪里真的还不够”。

另外，这一轮我又把 BN 诊断再往前推了一步，不只看“留一个 BN 公式出去”了，还额外加了两层：

- **BN family holdout**
  - selected model：`MAE = 1.5588`
  - screening model：`MAE = 1.4923`
  - dummy baseline：`MAE = 1.3255`
  - best candidate-compatible combo：`matminer_composition + torch_mlp_ensemble`，`MAE = 1.1382`
  - best overall combo：`matminer_composition_plus_structure_summary + torch_mlp`，`MAE = 1.0991`
- **BN vs non-BN stratified error**
  - selected model：`BN MAE = 1.7908`，`non-BN MAE = 0.5670`，误差比约 `3.16x`
  - screening model：`BN MAE = 1.5004`，`non-BN MAE = 0.5830`，误差比约 `2.57x`
  - `matminer_composition + torch_mlp_ensemble`：`BN MAE = 1.2128`，`non-BN MAE = 0.6697`，误差比约 `1.81x`
  - dummy baseline：`BN MAE = 1.3251`，`non-BN MAE = 1.1162`

这两层的意义现在变成两件事同时成立：

- 我们现在能更明确地说，BN 难不是偶然；
- 问题不只是在某几个 BN 公式上，而是在 BN 子域整体上都更难；
- 同时，第一条真正有希望的 BN candidate-compatible neural 路线已经出现了；
- 但目前真正被“完全解决”的仍然不是 BN generalization 本身，而只是从“全输 dummy”推进到“至少已有一条严格 BN 诊断下能打赢 dummy 的现代模型路线”。

### 2.3 这次新补上的 uncertainty / abstention / decision-policy，到底补到了哪里

这次真正有价值的新增，不是再多做一个 shortlist，而是把老师最容易追问的三件事落成了 artifact：

- `demo_candidate_ranking_uncertainty.csv`
  - 写出每个候选的 `predicted_band_gap_mean/std`
  - `rank_mean/std/min/max`
  - `top-3 / top-5 / top-10` 入选频率
  - `abstain_flag`
  - `reason_for_abstention`
  - `final_action_label`
- `bn_candidate_compatible_evaluation.csv`
  - 专门回答“真正能给无结构候选公式打分的组合，在 BN-slice 上到底怎么样”

当前读法很明确：

- general ranking 和 BN-centered ranking 的相关性仍然不低，但差异已经比以前更明显
  - top-3 overlap = `2/3`
  - top-5 overlap = `3/5`
  - top-10 overlap = `9/10`
  - Spearman = `0.9208`
  - Kendall tau = `0.7600`
- decision policy 仍然明显偏保守
  - `25` 个候选里有 `21` 个直接进入 abstain
- 当前真正**非 abstain 且非化学合理性拒绝**的正向 follow-up，只剩：
  - `BCN2`
  - `SiBN2`
  - `Si2BN`
- 更关键的是：
  - `bn_candidate_compatible_evaluation.csv` 当前共有 `11` 行 screening-compatible 视图
  - 最强 candidate-compatible 组合已经变成 `matminer_composition + torch_mlp_ensemble`
  - 它在 BN formula holdout 和 BN family holdout 上都已经打赢 global dummy baseline

这意味着这次更新不是把结果“说得更漂亮”，而是把“什么时候应该闭嘴、什么时候只能保守跟进”明确写出来了。

另外，这次还新增了一个更适合给导师直接看的总表：

- `bn_evaluation_matrix.csv`

它把：
- BN formula holdout
- BN family holdout
- BN vs non-BN grouped stratified error

放在一个矩阵里统一对照。这样之后再讲 BN，不需要在多个 CSV 之间来回跳，叙事会清楚很多。

## 3. 现在比旧版本真正强在哪里

如果从“给导师看是否更像博士项目”这个角度讲，我认为当前版本主要强在下面三点。

### 3.1 不再混淆不同问题

现在项目会明确区分：

- **overall evaluation best**
- **formula-only screening best**
- **BN-slice best**

这三个角色现在已经不是一个东西。

这非常重要，因为以前最容易被质疑的地方之一，就是把“整体 benchmark 最优”偷偷当成“候选筛选最优”甚至“BN 方向也最优”。

现在这个混淆基本已经被拆开了。

### 3.2 不再只会给一个 top-k

现在系统不会只说“前 10 个候选是哪些”，而是会分层给出：

- 完整 raw ranking
- family-aware proposal shortlist
- 只保留真正 unseen formulas 的 extrapolation shortlist
- BN-centered alternative ranking
- prototype-grounded structure follow-up shortlist
- novelty-aware structure follow-up shortlist

这会让你和导师讨论时，不再只能围绕“这个公式分数高不高”转，而是可以讨论：

- 这个候选是 rediscovery 还是 extrapolation
- 它是不是 BN family 里合理的一支
- 它有没有足够的 BN-local 支持
- 它是不是更适合进入下一步结构工作流

### 3.3 项目开始具备“后续研究骨架”

以前这个项目更像一个终点停在 ranking 的 demo。

现在它已经形成了一个更完整的骨架：

- 数据标准化
- 分层 benchmark
- BN-focused 诊断
- 多层 candidate screening
- structure follow-up planning

这意味着后续如果要继续推进：

- BN-centered modeling
- 更强的 BN-aware 决策层
- 真实结构生成 / relaxation / stability validation

都有了一个比较清晰的落点，而不是还停在“先排个名再说”。

## 5. 我对当前阶段的判断

如果要直接回答“这个版本是不是值得重新发给导师看”，我的判断仍然是：

**值得。**

原因不是它已经做完了，而是它已经明显从：

- 一个泛化的工程 demo

推进到了：

- 一个边界更清楚、对批评更有回应能力、也更像博士研究入口的 BN-themed PoC。

更直白一点说：

- 它现在比旧版本更容易得到导师的“认可你们确实在认真修方法”的评价；
- 但它还不足以让一个非常严厉的导师直接认为“核心科学问题已经解决”。

所以当前最合适的定位是：

**这是一个明显进步、值得重新汇报、但仍需继续攻克 BN-centered generalization 与真实结构验证的版本。**

## 6. 当前最值得继续推进的方向

从当前代码和 artifact 看，接下来最值得继续做的不是再堆报告字段，而是继续强化下面两类问题至少一类：

1. **BN-centered modeling / decision layer**
   - 让 BN 不只是一个解释和诊断视角，而更深地进入模型或决策逻辑本身。

2. **真实结构后续验证**
   - 把当前的 structure seed / job plan / queue，推进到真实 structure workflow，而不是停留在 planning artifact。

如果这两条线继续推进，这个项目才更可能从“方法学上更像研究”继续走向“研究结论也开始站得住”。
