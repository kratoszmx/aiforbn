# 项目汇报

## 项目题目

**AI for BN：一个引入轻量 structure-aware evaluation，并为 formula-only screening 增加化学可信度层与 novelty / rediscovery 透明分层的 band gap 预测 PoC**

---

## 1. 当前结论

本轮工作的核心不是把分数包装得更好看，而是把系统叙事和真实执行路径对齐。

当前系统已经完成：

- 使用 JARVIS / 2DMatPedia `twod_matpd` 构建标准化二维材料数据表；
- 采用 grouped-by-formula 的 train / val / test 评估协议；
- 在一个很小的 `{feature_set} x {model_type}` 空间上做 validation selection；
- 同时保留 composition-only 控制组，并新增 lightweight structure-aware 路径；
- 在测试集上输出完整 benchmark，并保留 `dummy_mean` baseline；
- 对 formula-only toy candidate space 做 ranking；
- 对 formula-only toy candidate space 增加 lightweight chemical-plausibility annotation 与显式 top-k 决策；
- 在 ranking artifact 中明确区分 best overall evaluation combo 与 best formula-only screening combo；
- 对候选显式区分 train+val rediscovery、held-out-known formula 与 formula-level extrapolation；
- 输出结构化 artifact，并可通过 notebook / Streamlit 展示。

因此，当前项目状态最准确的概括是：

**工程 PoC 成立；评估与筛选之间的逻辑边界已被显式写清；系统已经进入“轻量 structure-aware evaluation + 诚实 formula-only screening fallback”的阶段。**

但必须强调：

**这仍然不是可信的 structure-aware screening system，更不是候选发现结论。**

---

## 2. 这轮之后，主要解决了什么问题

### 2.1 评估协议继续保持在 grouped-by-formula

数据集中共有：

- 6351 条记录
- 4381 个唯一 `formula`

因此若按随机行切分，formula leakage 风险会非常明显。

当前默认 split 仍为：

**按 `formula` 分组切分。**

最新验证运行中：

- train：5101 rows / 3505 formulas
- val：618 rows / 438 formulas
- test：632 rows / 438 formulas
- `train_val` / `train_test` / `val_test` overlap 全部为 **0**

这意味着当前 test 指标不再建立在默认 formula 泄漏之上。

### 2.2 特征空间已从“双 composition-only”升级为“三路径搜索”

当前默认候选特征集为：

1. `basic_formula_composition`
   - 7 维手写基础控制组

2. `matminer_composition`
   - 19 维 composition-only 特征
   - 来自 `Stoichiometry` 与选定 `Magpie` 元素属性统计

3. `matminer_composition_plus_structure_summary`
   - 上述 19 维 + 11 个结构摘要列
   - 结构摘要来自 cached raw JARVIS `atoms` / `lattice` 信息

这里的结构路径仍然是轻量级摘要，不是图网络或高阶结构表征。
但它已经足以让系统第一次出现一个方法学上更合理的分叉：

- overall evaluation 可以选 structure-aware 路径；
- formula-only candidates 不能。

### 2.3 validation selection 现在真正区分了 evaluation 与 screening

当前验证集选择不是只选模型，而是在以下 6 个组合中选：

- `basic_formula_composition + hist_gradient_boosting`
- `basic_formula_composition + linear_regression`
- `matminer_composition + hist_gradient_boosting`
- `matminer_composition + linear_regression`
- `matminer_composition_plus_structure_summary + hist_gradient_boosting`
- `matminer_composition_plus_structure_summary + linear_regression`

这轮最重要的结果是：

- **best overall evaluation combo**
  - `matminer_composition_plus_structure_summary + hist_gradient_boosting`
- **best formula-only screening combo**
  - `matminer_composition + hist_gradient_boosting`

两者不同。

这意味着当前系统已经不再犯下一个很常见的叙事错误：

**不会再把“测试集上最优的 structure-aware 组合”直接拿去给没有结构的候选公式打分。**

### 2.4 ranking artifact 现在能明确暴露这种 fallback

当前 `demo_candidate_ranking.csv` 中会写出：

- `ranking_feature_set`
- `ranking_model_type`
- `best_overall_evaluation_feature_set`
- `best_overall_evaluation_model_type`
- `screening_matches_best_overall_evaluation`
- `screening_selection_note`

因此读者可以直接看到：

- 当前排序是否复用了 overall best combo；
- 如果没有，是因为候选缺少结构而进行了 candidate-compatible fallback。

### 2.5 候选空间现在多了一层基础化学可信度筛选

本轮没有假装解决稳定性、结构或可合成性问题，而是只加了一层**轻量、可解释、公式级**的 chemical plausibility 检查：

- 方法：`pymatgen` oxidation-state guesses
- 目标：检查候选公式是否存在基础的 charge-balanced common oxidation-state assignment
- 作用方式：
  - 保留原始 25 个 Group 13 / 15 source-space 公式；
  - 为每个公式写出 `chemical_plausibility_pass`、guess 数量与 top guess；
  - final artifact 中显式写 `screening_selected_for_top_k` 与 `screening_selection_decision`

这轮验证下：

- 25 个 toy source-space 候选中，**21 个通过**
- **4 个失败**
- 当前失败公式为：`AlBi`, `GaBi`, `InBi`, `TlBi`

因此当前 screening 行为已经从：

- “透明 toy grid 直接打分”

升级为：

- “透明 toy grid + 基础化学可信度注释 + 对未通过公式的显式降级”

### 2.6 uncertainty 与 overlap 风险继续被显式暴露

当前 ranking 仍保留：

- `chemical_plausibility_pass`
- `chemical_plausibility_guess_count`
- `chemical_plausibility_primary_oxidation_state_guess`
- `chemical_plausibility_note`
- `ensemble_predicted_band_gap_mean`
- `ensemble_predicted_band_gap_std`
- `ensemble_member_count`
- `ranking_score`
- `seen_in_dataset`
- `dataset_formula_row_count`
- `seen_in_train_plus_val`
- `train_plus_val_formula_row_count`

所以当前 ranking 的正确定位仍然是：

- sanity-check ranking
- pipeline-level demo

而不是：

- discovery claim
- experimentally actionable screening result

### 2.7 候选新颖性现在被显式分层，而不是继续混在同一个 top-k 叙事里

这是本轮最直接回应“当前 candidate space 更像常识回放而不是真正 screening”的改进。

系统现在会把 25 个 toy candidates 明确分成三类：

- `train_plus_val_rediscovery`
  - 已在 `train+val` 中出现，本质上是 in-domain replay
- `held_out_known_formula`
  - 不在 `train+val`，但在 dataset 其他部分出现过，本质上是 held-out-known formula
- `formula_level_extrapolation`
  - 整个 dataset 中都未出现过，才算公式级外推

最新 artifact 的真实计数为：

- `train_plus_val_rediscovery`：**23**
- `held_out_known_formula`：**1**，即 `AlP`
- `formula_level_extrapolation`：**1**，即 `BBi`

标准 top-k 20 的构成为：

- rediscovery：**19**
- held-out-known：**1**
- formula-level extrapolation：**0**

因此，这轮工作虽然没有把 toy candidate space 变成真正 discovery space，但至少做到了两点：

1. 不再把 rediscovery、held-out-known 和真正 dataset 未见公式混成同一种“候选”；
2. 明确暴露当前 top-k 基本仍然是已知公式重放，而不是把它包装成外推筛选能力。

## 3. 当前系统设定

### 3.1 数据与任务

- 数据集：`twod_matpd`
- 数据源：JARVIS / 2DMatPedia
- 目标列：`band_gap`
- 总数据量：6351 rows
- 唯一 `formula`：4381
- BN 主题切片：12 rows / 10 unique BN formulas

BN 当前仍然主要是：

- 主题切口；
- 结果展示焦点；
- 候选空间叙事入口。

训练主体仍然是整个二维材料表，而不是 BN-only 学习。

### 3.2 模型流程

当前默认流程为：

1. 构造 grouped split；
2. 构造三套 feature tables；
3. 在 validation 上选择 best overall evaluation combo；
4. 同时选出 best formula-only screening combo；
5. 用 `train + val` 重训 overall best combo，并在 test 上评估；
6. 对所有组合与 `dummy_mean` 生成 benchmark；
7. 用公式兼容的 screening combo 生成候选排名；
8. 将 honesty-oriented metadata 写入 artifact。

### 3.3 当前 artifact

主 artifact 包括：

- `metrics.json`
- `benchmark_results.csv`
- `experiment_summary.json`
- `predictions.csv`
- `bn_slice.csv`
- `demo_candidate_ranking.csv`
- `parity_plot.png`

### 4.4 对这些数字的克制解读

可以成立的结论有三点：

1. lightweight structure-aware 路径相对最佳 formula-only 路径有**小幅但一致**的提升。
   - test MAE 从 `0.5822` 降到 `0.5568`
   - 绝对改善约 `0.0254`
   - 相对改善约 `4.4%`
2. richer composition representation 明显优于最弱的手写控制组。
3. 当前系统明显优于 `dummy_mean`，说明并非完全无信号。

但不能夸大的地方同样明确：

- 这不是巨大跃升；
- 这不是成熟 structure-aware 建模；
- 这不足以支撑 candidate discovery claim。

### 4.5 当前 demo candidate ranking 的解读

当前前列结果仍以常见 III-V 风格组合为主，例如：

- `BN`
- `AlN`
- `GaN`
- `AlP`
- `InN`

而在当前 artifact 中，以下公式被明确标成：

- `screening_selection_decision = failed_chemical_plausibility`

即：

- `InBi`
- `TlBi`
- `GaBi`
- `AlBi`

同时，当前 novelty 分层给出的解释是：

- `AlP` 不是新公式，而是 **held-out-known formula**；
- 当前唯一 **formula-level extrapolation** 是 `BBi`；
- `BBi` 虽然通过了轻量 chemical plausibility，但 `ranking_rank = 21`，没有进入 top-k。

这更像：

- 模型没有明显跑偏；
- demo ranking 行为可解释；
- 常识回放得到了一定程度保留；
- 候选空间开始有了最基本的化学式可信度边界；
- 系统终于能诚实区分“重放已知公式”和“真正 dataset 未见公式”。

这并不等于：

- 模型具有可信的新候选发现能力。

## 5. 当前方法学判断

### 5.1 已经站得住的部分

- grouped-by-formula 评估协议；
- candidate feature/model benchmark；
- cached raw JARVIS 数据驱动的轻量 structure-aware 路径；
- overall evaluation 与 formula-only screening 的显式分离；
- ranking 中的不确定性 heuristic 与 overlap 风险暴露；
- rediscovery / held-out-known / formula-level extrapolation 的显式新颖性分层。

### 5.2 仍然站不住的部分

- 结构路径仍只是摘要级，不是强结构表征；
- formula-only candidates 仍然无法进行真正的 structure-aware screening；
- 当前 toy candidate space 仍然几乎全是已知公式，真正 dataset 未见的只有 `BBi` 一个；
- BN slice 太小，无法支撑“面向 BN 的方法优势”；
- toy candidate space 仍无结构、稳定性、可合成性约束；
- disagreement 仍只是 heuristic，不是 calibrated uncertainty；
- train / inference 仍未解耦。

因此，此阶段最准确的评价应当是：

**PoC 已经从 stronger composition-only baseline，进一步推进到了 lightweight structure-aware evaluation with honest formula-only screening fallback。**

但它依然只是研究舞台的一部分，不是研究结论本身。

---

## 6. 下一步优先级

### Priority 1: 强化真正的 structure-aware 表示

- 不停留在结构摘要层；
- 引入更像材料建模而不是表格工程的结构表示。

### Priority 2: 提升 candidate space credibility

- 先把 candidate source space 从“几乎全是已知公式”推进到包含更多真正 dataset 未见公式；
- 再给候选空间加入结构、稳定性、形成能或可合成性约束；
- 进一步降低“常识回放”的解释空间。

### Priority 3: 升级不确定性处理

- 从小模型池 disagreement heuristic 走向更可辩护的 uncertainty estimation。

### Priority 4: 解耦 train / inference

- 让当前 PoC 从“线性 demo 脚本”进一步走向可复用原型。
