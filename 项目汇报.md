# 项目汇报

## 项目题目

**AI for BN：一个面向 BN 主题的组成级 band gap 预测与 demo candidate ranking PoC**

---

## 1. 当前结论

本轮工作的核心不是“把分数再抬一点”，而是修正最明显的方法学漏洞，并让项目叙事与真实实现对齐。

当前系统已经完成：

- 从 JARVIS / 2DMatPedia 拉取和规范化二维材料数据；
- 基于化学式构造基础组成特征；
- 采用 grouped-by-formula 的 train / val / test 评估协议；
- 在很小的候选模型集合上用验证集做选择；
- 在测试集上与 `dummy_mean` 做 benchmark；
- 对一个明确标注为 toy/demo 的候选空间做排序；
- 输出结构化 artifact，并通过 Streamlit / notebook 展示。

因此，项目现在的状态可以概括为：

**工程 PoC 成立，方法学诚实度较上一轮明显提升，但研究结论仍然只能算 weak baseline evidence。**

---

## 2. 这轮升级主要解决了什么问题

### 2.1 修复了默认评估协议中的 formula leakage 风险

上一轮最严重的问题是：

- 数据集中共有 6351 条记录，但只有 4381 个唯一 `formula`；
- 若采用随机行级切分，同一 `formula` 可能同时出现在训练集和测试集；
- 对当前这种 formula-derived composition baseline，这会显著高估泛化能力。

本轮默认 split 已改为：

**按 `formula` 分组切分。**

最新一次验证运行中：

- train：5101 rows / 3505 formulas
- val：618 rows / 438 formulas
- test：632 rows / 438 formulas
- `train_val` / `train_test` / `val_test` 的 formula overlap 均为 **0**

这意味着当前 test 指标至少不再建立在默认的 formula 泄漏之上。

### 2.2 增加了最低限度但必要的 baseline comparison

上一轮另一个明显短板是：

- 只有单模型分数；
- 没有 naive baseline；
- 因而很难解释“这个分数到底意味着什么”。

本轮新增了 `benchmark_results.csv`，默认在测试集上对比：

- `linear_regression`
- `hist_gradient_boosting`
- `dummy_mean`

这至少能回答两个基础问题：

1. 当前模型是否优于平均值猜测；
2. 当前树模型是否优于简单线性模型。

### 2.3 验证集现在真正用于模型选择

当前没有做复杂调参，只在一个很小的候选集合里选择模型：

- `linear_regression`
- `hist_gradient_boosting`

选择指标为 validation MAE。

最新一次运行中：

- `linear_regression` validation MAE = **0.9976**
- `hist_gradient_boosting` validation MAE = **0.9469**

因此，最终被选中并在 `train + val` 上重训的是：

- **`hist_gradient_boosting`**

### 2.4 配置与实现描述已经对齐

上一轮文档与配置中存在“名义能力大于真实能力”的问题，最典型的是 `use_matminer = True` 但代码路径并未真正接入 matminer。

本轮已经显式改为：

- `feature_set = basic_formula_composition`
- `matminer_enabled = False`

因此，当前系统应被准确描述为：

**composition-only baseline，而不是 matminer-enhanced baseline。**

### 2.5 候选排序的命名已收敛到 demo 语义

当前候选空间仍然只是：

- Group 13: `B, Al, Ga, In, Tl`
- Group 15: `N, P, As, Sb, Bi`
- 两两组合，共 25 个 formula

这类空间不足以支撑严肃 screening claim。

因此本轮做了两件事：

- 候选空间名称改为 `toy_iii_v_demo_grid`
- 排序输出文件改为 `demo_candidate_ranking.csv`

同时在文件元数据中明确写入：

- `candidate_space_kind = toy_demo`
- `candidate_generation_strategy = group13_group15_cartesian_product`
- 无稳定性、结构、合成可行性约束

## 3. 当前系统设定

### 3.1 数据与任务

- 数据集：`twod_matpd`
- 数据源：JARVIS / 2DMatPedia
- 目标列：`band_gap`
- 总数据量：6351 rows
- 唯一 `formula`：4381
- BN 主题切片：12 rows，10 个唯一 BN formulas

需要强调：

- BN 目前更多是**主题切口与展示聚焦**；
- 当前训练主体仍然是整个二维材料表，而不是 BN-only 数据。

### 3.2 特征

当前特征为 `basic_formula_composition`，包括：

- `n_elements`
- `sum_z`
- `max_z`
- `min_z`
- `mean_z`
- `contains_B`
- `contains_N`

这是一组明确的 formula-derived hand-crafted features，不含结构信息，也不含 matminer 特征。

### 3.3 模型流程

当前默认流程为：

1. 构造 grouped split；
2. 在 validation 上比较 `linear_regression` 与 `hist_gradient_boosting`；
3. 选择更优模型；
4. 用 `train + val` 重训被选模型；
5. 在 test 上报告最终指标；
6. 输出 benchmark artifact；
7. 用被选模型对 toy/demo 候选空间打分。

### 3.4 Artifact

当前主 artifact 包括：

- `metrics.json`
- `benchmark_results.csv`
- `experiment_summary.json`
- `predictions.csv`
- `bn_slice.csv`
- `demo_candidate_ranking.csv`
- `parity_plot.png`

## 4. 最新一次验证运行结果

### 4.1 被选模型的测试集结果

被 validation 选中的模型是：

- `hist_gradient_boosting`

其测试集指标为：

- **MAE = 0.9114**
- **RMSE = 1.1813**
- **R² = 0.2358**

### 4.2 benchmark 对照

同一轮测试集 benchmark 结果为：

| model | role | MAE | RMSE | R² |
| --- | --- | ---: | ---: | ---: |
| `hist_gradient_boosting` | selected_model | 0.9114 | 1.1813 | 0.2358 |
| `linear_regression` | candidate_model | 1.0056 | 1.2660 | 0.1223 |
| `dummy_mean` | dummy_baseline | 1.1118 | 1.3531 | -0.0026 |

从这个表里可以得出较克制但成立的解释：

- 当前 baseline 至少优于 naive mean predictor；
- 非线性模型在当前特征下优于简单线性模型；
- 但整体 R² 仍然偏低，只能说明存在有限可学信号，不能据此做强筛选结论。

### 4.3 demo candidate ranking

当前 `demo_candidate_ranking.csv` 的前列结果仍然以常见 III-V 风格组合为主，例如：

- `BN`
- `AlN`
- `GaN`
- `AlP`
- `GaP`

这一结果更适合解释为：

- baseline 没有完全失真；
- ranking 流程已可运行；
- 输出至少不是一眼荒谬。

它不应被解释为：

- 模型已经具备可信候选发现能力；
- top candidates 已经可以直接指导实验。

## 5. 当前方法学判断

### 5.1 现在已经站得住的部分

- 默认 leakage 风险已被显式压下去；
- benchmark 不再缺失；
- validation split 不再只是摆设；
- 配置与实现描述已经一致；
- candidate ranking 的叙事已经收敛到 demo 语义。

### 5.2 现在仍然站不住的部分

- 当前仍是 composition-only baseline；
- BN slice 规模过小，尚不能支持“专门面向 BN 的方法优势”；
- toy/demo candidate space 缺乏结构、稳定性、可合成性约束；
- 还没有 uncertainty-aware ranking；
- 还没有 train / inference 的真正解耦接口。

因此，这一阶段最准确的评价仍然是：

**PoC 已经从“能跑”进化到“更可辩护地能跑”，但真正有研究力度的方法工作还没有完成。**

---

## 6. 下一步优先级

若继续往科研原型推进，建议优先级保持为：

### Priority 1: 提升表示能力，但保持叙事诚实

- 真正接入 matminer composition features，或引入结构级特征；
- 同时保留当前 `basic_formula_composition` 作为弱基线对照。

### Priority 2: 提升 candidate space credibility

- 对候选空间加入最基础的稳定性、结构合理性或元素组合约束；
- 把“toy demo ranking”逐步推向“有限但可信的 screening setup”。

### Priority 3: 加入 uncertainty-aware ranking

- 让排序结果不只给 point estimate；
- 至少给出粗粒度不确定性信号。

### Priority 4: 解耦 train / inference

- 提供统一的公式推理入口；
- 让离线训练与候选打分能够更清晰地分离。

---

## 7. 总结

本轮之后，这个项目最重要的变化不是“分数更高了一点”，而是：

- 默认评估协议更可信；
- benchmark 更完整；
- 候选叙事更收敛；
- 配置与实现更诚实；
- artifact 更适合向导师或合作者解释当前系统到底做了什么。

所以现在可以比较稳妥地说：

**该项目已经完成了一条更可辩护的 AI-for-materials PoC 工作流。**

但同样需要明确：

**它目前仍然只是 weak baseline + demo ranking，而不是可直接主张材料发现能力的研究系统。**
