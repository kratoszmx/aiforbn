# 项目汇报

## 本版更新说明（供快速浏览）

若只看这次版本相较于旧版真正新增了什么，可以先看这四点：

1. **评估协议更严格了**
   - 主评估继续使用 `group_by_formula`，避免同一公式泄漏到不同 split。
   - 新增 `grouped-by-formula` 的 5-fold robustness benchmark，不再只讲一次 holdout 的结果。
   - 新增 **BN-slice leave-one-BN-formula-out**、**BN family holdout** 与 **BN vs non-BN stratified error** 三层诊断，并加入 `dummy_mean` baseline，专门检查 BN 方向的外推是否真的成立。

2. **候选空间不再是泛泛的 toy grid**
   - 默认候选空间已重构为 **BN-anchored formula-family grid**，围绕 `BN / BCN / BC2N / Si2BN` 一类 BN 相关母题组织，而不是旧的泛化公式笛卡尔积。

3. **候选筛选的解释层次更完整了**
   - ranking 现在不是单一 top-k，而是同时输出：
     - raw ranking
     - family-aware proposal shortlist
     - formula-level extrapolation shortlist
     - BN-centered alternative ranking
     - ranking uncertainty / abstention / decision-policy artifact
   - 同时加入 `BN-local support`、`BN analog evidence`、`BN analog validation penalty`、`grouped robustness penalty`、`top-k stability` 等解释层。

4. **项目已经从“公式排序”桥接到“后续结构跟进计划与 first-pass prototype 执行”**
   - 当前系统会额外生成 structure seed / handoff / reference record / job plan / first-pass queue / follow-up shortlist，以及 first-pass execution 的 JSON / CSV / `.cif` artifact。
   - 这仍不是结构生成或结构验证，但已经把“排完名之后下一步怎么做”，推进到了“先把哪几条 prototype 真正 materialize 出来”。

## 1. 任务定义与当前范围

### 1.1 当前任务

项目当前聚焦于两个层面：

1. **监督学习层面**
   - 使用公开二维材料数据做 `band_gap` 回归预测。

2. **候选筛选层面**
   - 在一个小规模、BN 主题明确的公式候选空间上，做**诚实的公式级 screening 演示**。

### 1.2 数据来源与规模

当前使用的数据为 JARVIS / 2DMatPedia 的 `twod_matpd` 缓存数据，当前主流程对应的规范化数据规模为：

- `6351` 条记录
- `4381` 个唯一公式
- BN 主题切片：`12` 条记录，`10` 个唯一 BN 公式

### 1.3 需要强调的边界

本项目现在解决的是：

- 公开二维材料数据上的 band-gap 预测基线；
- BN 主题下一个小型候选空间的演示性排序；
- 候选从 ranking 过渡到后续结构工作流的桥接。

本项目**没有**解决的是：

- 真实结构生成；
- 结构松弛后的稳定性验证；
- 合成可行性判断；
- 可直接宣称“发现新 BN 材料”的证据闭环。

## 2. 当前方法实现

### 2.1 数据划分：`group_by_formula`

当前主评估协议继续使用按公式分组切分，而不是按行随机切分。当前 split 为：

- train：`5101` rows / `3505` formulas
- val：`618` rows / `438` formulas
- test：`632` rows / `438` formulas

并且：

- `train-val` overlap = `0`
- `train-test` overlap = `0`
- `val-test` overlap = `0`

这意味着当前 test benchmark 不建立在 formula leakage 之上。

### 2.2 四条特征路径

当前系统保留四条特征路径，避免只讲一个模型结果：

1. `basic_formula_composition`
   - 7 维手写组成控制特征
   - 作用：最低限度、最容易解释的 control baseline

2. `matminer_composition`
   - 19 维 composition-only 特征
   - 作用：当前公式级候选筛选的主要 hand-crafted descriptor 路径

3. `fractional_composition_vector`
   - 118 维原始元素分数向量
   - 作用：为 learned neural composition baseline 提供更直接的原始组成输入

4. `matminer_composition_plus_structure_summary`
   - `matminer_composition` + 11 个轻量结构摘要特征
   - 作用：当前总体评估里的 structure-aware 路径

这里的结构信息仍然只是轻量 summary，不是图网络，也不是显式结构生成模型。

### 2.3 模型空间

当前主流程保留两类传统控制模型，并新增一条低依赖 neural baseline family：

- `hist_gradient_boosting`
- `linear_regression`
- `torch_mlp`
- `torch_mlp_ensemble`

这里的 `torch_mlp / torch_mlp_ensemble` 不是为了把项目包装成“已经用了最强 SOTA”，而是为了让 repo 在不依赖老旧第三方 CrabNet 包的前提下，先获得一条真实可验证、并且能进一步做 seed-stability 加强的 modern neural baseline。

设计意图仍然是：

- 对照清晰
- 结果易解释
- 组合空间可验证
- 在保持主流程可控的前提下，逐步引入 modern model

### 2.4 evaluation 与 screening 已显式分离

当前代码中，validation selection 不是只选一个“最优模型”，而是明确区分两个角色：

- **best overall evaluation combo**
- **best candidate-compatible screening combo**

当前自动选择结果为：

- **overall evaluation 最优**
  - `matminer_composition_plus_structure_summary + hist_gradient_boosting`
- **formula-only screening 最优**
  - `matminer_composition + hist_gradient_boosting`

这一步很关键，因为它避免了一个常见但不严谨的做法：

**不能把需要结构输入的最优评估模型，直接拿去给没有结构的候选公式打分。**

### 2.5 候选空间：BN-anchored family grid

当前默认候选空间已经改为 **BN-anchored formula-family grid**，总计 `25` 个候选公式。

它不再是泛化的 III-V toy grid，而是围绕下列 BN 相关母题构造：

- `BN` 二元 anchor
- `BCN / BC2N` 相关母题
- `Si2BN` 一类数据集中真实出现过的 BN ternary motif

当前 family 分布为：

- `bn_binary_anchor`: 1
- `group14_bn_111_family`: 4
- `group14_bn_211_family`: 4
- `group14_bn_121_family`: 4
- `group13_bn_111_family`: 4
- `group13_bn_211_family`: 4
- `group13_bn_121_family`: 4

因此，现在不是“先生成 generic 候选，再拿 BN 做包装”，而是：

**候选空间本身就是按 BN family 假设来组织的。**

### 2.6 当前 ranking 不是单一分数，而是分层解释

当前 `demo_candidate_ranking.csv` 不只是给出 `predicted_band_gap` 和 `ranking_rank`，还同时保留以下解释层：

- chemical plausibility
- global domain support
- BN-local support
- BN analog evidence
- BN analog validation penalty
- BN-local band-gap alignment
- grouped robustness penalty
- novelty bucket
- proposal shortlist decision
- extrapolation shortlist decision

这意味着当前 artifact 已经从“简单分数排序”变成“带边界条件和证据注释的排序”。

## 3. 已验证结果

### 3.1 标准 test benchmark

当前主流程生成的 `metrics.json` 与 `benchmark_results.csv` 给出的核心结果如下。

### 当前 selected model（overall evaluation）

- feature set: `matminer_composition_plus_structure_summary`
- model: `hist_gradient_boosting`
- `MAE = 0.5568`
- `RMSE = 0.8218`
- `R² = 0.6302`

### 当前最佳 formula-only candidate-compatible 路径

- feature set: `matminer_composition`
- model: `hist_gradient_boosting`
- `MAE = 0.5822`
- `RMSE = 0.8611`
- `R² = 0.5939`

### dummy baseline

- model: `dummy_mean`
- `MAE = 1.1118`
- `RMSE = 1.3531`
- `R² = -0.0026`

### 标准 test benchmark 的直接结论

1. 当前系统在一般二维材料 band-gap 预测上，已经明显优于 dummy baseline。
2. 轻量 structure-aware 路径相较最佳 formula-only 路径，仍然存在**小幅但一致**的优势：
   - test MAE 从 `0.5822` 降到 `0.5568`
   - 绝对改善约 `0.0254`
   - 相对改善约 `4.4%`
3. 因此，“结构摘要在总体评估里有增益”这件事，是可以成立的。

但这并不意味着：

- 结构路径已经足够强；
- 或者它可以被直接用于无结构候选筛选。

### 3.2 grouped-by-formula robustness benchmark

单次 holdout 的好结果还不够，因此当前代码又加入了 5-fold grouped robustness benchmark。

核心结果为：

| 组合 | MAE_mean | MAE_std | 说明 |
|---|---:|---:|---|
| `matminer_composition_plus_structure_summary + hist_gradient_boosting` | `0.6050` | `0.0176` | 当前 overall evaluation 路径 |
| `matminer_composition + hist_gradient_boosting` | `0.6321` | `0.0192` | 当前 formula-only screening 路径 |
| `basic_formula_composition + hist_gradient_boosting` | `1.0116` | `0.0173` | 低维控制组 |
| `dummy_mean` | `1.1527` | `0.0233` | 无特征 baseline |

这里可以得出两个比较稳的结论：

1. 当前系统的主路径优势，不只是单次 holdout 的偶然结果。
2. lightweight structure-aware 路径相对最佳 formula-only 路径的优势，在 grouped robustness 下仍然存在。

但仍需强调：

- 这只是 grouped internal validation；
- 不是外部数据集验证；
- 更不是 prospective validation。

### 3.3 BN-slice benchmark：当前最关键也最诚实的诊断

这是当前版本最重要的新增部分之一。

原因是：在标准 grouped split 下，当前 BN slice 的分布为：

- BN rows in train = `12`
- BN rows in val = `0`
- BN rows in test = `0`

也就是说，**如果只看标准 test benchmark，实际上并没有真正检验到 BN 公式的外推能力。**

因此当前系统新增了 **leave-one-BN-formula-out** 的专门诊断，结果如下：

| 角色 / 组合 | MAE | R² | 结论 |
|---|---:|---:|---|
| selected model: `matminer_composition_plus_structure_summary + hist_gradient_boosting` | `1.6397` | `-0.2220` | 未优于 dummy |
| screening model: `matminer_composition + hist_gradient_boosting` | `1.6383` | `-0.2662` | 未优于 dummy |
| best BN candidate-compatible combo: `matminer_composition + torch_mlp_ensemble` | `1.1257` | `0.0030` | 已优于 dummy |
| best BN overall combo: `matminer_composition_plus_structure_summary + torch_mlp` | `1.0693` | `0.2248` | 明显优于 dummy |
| global dummy baseline | `1.3257` | `-0.1021` | 基础参照 |
| BN-local reference baseline (`bn_local_knn_mean`) | `2.2142` | `-0.7992` | 明显更差 |

### BN-slice benchmark 的结论必须如实写

这部分不能粉饰。当前最准确的结论是：

1. 在**一般二维材料预测**上，当前系统已经有可信的内部 benchmark 增益。
2. 但在**BN-centered leave-one-formula-out** 诊断上：
   - 当前 selected model 没有打赢 dummy；
   - 当前默认 screening model 也没有打赢 dummy。
3. 不过这次 modern-model wave 之后，已经出现了一个真正 candidate-compatible 且能在 BN-slice 上**明确打赢 dummy** 的 neural baseline：
   - `matminer_composition + torch_mlp_ensemble`
4. 进一步把 BN-slice 重新投影到**真正 screening-compatible 的组合**以后，`bn_candidate_compatible_evaluation.csv` 当前共有 `11` 行视图，其中最优 candidate-compatible 组合已不再输给 global dummy baseline。

因此当前版本最重要的研究性结论不是“BN 已经做得很好”，而是：

**当前系统已经把“整体二维材料预测能力”和“BN-centered generalization 能力”这两个问题明确拆开了，而且后者虽然仍是主要短板，但现在终于出现了一条在严格 BN 诊断下站得住的 candidate-compatible modern-model 路线。**

我认为这反而比旧版更适合导师阅读，因为它把真正的问题暴露出来了。

### 3.4 BN family holdout 与 BN stratified error：把“BN 更难”从直觉变成量化结论

只做 BN formula holdout 还不够，所以当前版本又往前补了两层。

#### BN family holdout

当前新增 `leave_one_bn_family_out` 结果如下：

| 角色 / 组合 | MAE | R² | 结论 |
|---|---:|---:|---|
| selected model: `matminer_composition_plus_structure_summary + hist_gradient_boosting` | `1.5588` | `-0.1338` | 未优于 dummy |
| screening model: `matminer_composition + hist_gradient_boosting` | `1.4923` | `-0.1362` | 未优于 dummy |
| best family-holdout candidate-compatible combo: `matminer_composition + torch_mlp_ensemble` | `1.1382` | `-0.0110` | 已优于 dummy |
| best family-holdout overall combo: `matminer_composition_plus_structure_summary + torch_mlp` | `1.0991` | `0.1966` | 明显优于 dummy |
| global dummy baseline | `1.3255` | `-0.1018` | 基础参照 |

这说明问题不只是“某几个 BN 公式恰好难”，而是**按 BN-local chemical family 聚合留出时，当前默认主模型仍然不稳**，但新的 neural baseline 已经给出了更有希望的替代路线。

#### BN vs non-BN stratified error

当前 grouped cross-validation 的分层误差如下：

| 角色 / 组合 | BN MAE | non-BN MAE | BN/non-BN MAE ratio |
|---|---:|---:|---:|
| selected model: `matminer_composition_plus_structure_summary + hist_gradient_boosting` | `1.7908` | `0.5670` | `3.16` |
| screening model: `matminer_composition + hist_gradient_boosting` | `1.5004` | `0.5830` | `2.57` |
| candidate-compatible neural baseline: `matminer_composition + torch_mlp_ensemble` | `1.2128` | `0.6697` | `1.81` |
| dummy baseline | `1.3251` | `1.1162` | `1.19` |

这层结果的意义非常直接：

1. 当前模型在一般二维材料上有效，但 **BN 子域明显更难**；
2. BN 误差放大不是体感，而是已经在 artifact 里被量化；
3. 不过新的 candidate-compatible neural baseline 已经把 BN/non-BN error ratio 从旧 screening combo 的 `2.57` 明显压低到了 `1.81`；
4. 因此当前更合适的项目表述仍然是 **BN-themed follow-up prioritization PoC**，不是 BN discovery/generalization system，但它已经不再只是纯传统模型控制组故事。

## 4. 候选筛选与输出层次

### 4.1 chemical plausibility

当前候选空间共 `25` 个公式，使用 `pymatgen` common oxidation-state guess 做轻量化学合理性筛查。

结果为：

- `23 / 25` 个候选通过
- `2 / 25` 个候选未通过：
  - `AlBN`
  - `TlBN`

这层不等于稳定性或可合成性判断，只是最基础的公式级 credibility boundary。

### 4.2 novelty layering

当前候选的 novelty 分布为：

- `train_plus_val_rediscovery`: `3`
  - `BN`
  - `BC2N`
  - `Si2BN`
- `held_out_known_formula`: `0`
- `formula_level_extrapolation`: `22`

这意味着当前候选空间已经不再主要由已知公式重放构成，而是以**公式级外推**为主。

但同样需要准确表述：

**这里的 novelty 只是 formula-level extrapolation，不是结构级或实验级 discovery。**

### 4.3 raw ranking 之外的多层 shortlist / control 输出

当前主流程不会只输出一个 top-k，而是至少输出五类 advisor-facing 结果：

1. **raw ranking**
   - 完整保留 25 个候选及其注释字段

2. **family-aware proposal shortlist**
   - 大小：`10`
   - 目的：限制单一 `candidate_family` 过度集中，避免 top list 被一个 family 垄断

3. **formula-level extrapolation shortlist**
   - 大小：`5`
   - 当前公式：
     - `BCN2`
     - `BCN`
     - `AlBN2`
     - `Al2BN`
     - `Ge2BN`
   - 目的：把“真正未见过的公式级外推候选”单独拉出来

4. **BN-centered alternative ranking**
   - 由 BN-slice benchmark 里当前最优的 candidate-compatible 组合驱动
   - 当前选中：`matminer_composition + torch_mlp_ensemble`
   - 作用：作为 BN-centered sensitivity / control view，而不是替代默认 general ranking

5. **ranking uncertainty / abstention / decision-policy artifact**
   - artifact：`demo_candidate_ranking_uncertainty.csv`
   - 会显式写出：
     - `predicted_band_gap_mean/std`
     - `rank_mean/std/min/max`
     - `top-3 / top-5 / top-10` selection frequency
     - `abstain_flag`
     - `reason_for_abstention`
     - `final_action_label`
   - 作用：把“哪些候选只是排在前面”和“哪些候选在当前证据下真的值得继续推”拆开

6. **BN evaluation matrix artifact**
   - artifact：`bn_evaluation_matrix.csv`
   - 作用：把 BN formula holdout、BN family holdout、BN vs non-BN stratified error 放到一个对照矩阵里，方便导师直接看“selected model / screening model / dummy / best candidate combo”在 BN 问题上的差别

### 当前 BN-centered control view 的意义

当前 BN-centered ranking 与 general ranking 的 top-20 成员重合度为 `19/20`，说明：

- 当前 top candidate set 仍然高度相关；
- 但新 neural BN-centered control view 已经不再只是“换个排序标签”，而是真的带来了更明显的前列重排。

当前更关键的稳定性数字是：

- top-3 overlap = `2/3`
- top-5 overlap = `3/5`
- top-10 overlap = `9/10`
- Spearman = `0.9208`
- Kendall tau = `0.7600`

它带来的有意义变化包括：

- `BN` 升到第 `2` 位附近的前列位置
- `SnBN2`、`InBN2` 这类候选在 BN-centered view 里被明显前移
- `BCN2` 虽然仍在前列，但不再像 general ranking 那样稳定占据第 `1`

因此，当前 BN-centered ranking 的价值不是“换一个模型名字”，而是：

**用严格 BN benchmark 选出的 candidate-compatible modern neural model，对默认 ranking 做更有分辨力的 BN-centered sensitivity / control 对照。**

### 4.4 这次 uncertainty / decision-policy 层，实际回答了什么问题

这次新增的重点，不是再做一个更花哨的 shortlist，而是把老师批评的“不确定性、abstention、decision policy 不清楚”落成了真实 artifact。

当前 `demo_candidate_ranking_uncertainty.csv` 与 `experiment_summary.json` 给出的关键信息包括：

- ranking-stability source count = `28`
- top-3 overlap = `2/3`
- top-5 overlap = `3/5`
- top-10 overlap = `9/10`
- Spearman rank correlation = `0.9208`
- Kendall tau = `0.7600`

这说明：

- general ranking 和 BN-centered ranking **并不是完全一样**；
- 但差异更多体现为**前列候选内部重排**，而不是整个候选集合完全翻盘。

更重要的是，当前 decision policy 明显是**保守型**的：

- `25` 个候选里，`21` 个直接被标记为 abstain
- 默认 decision policy 仍然非常保守，即使 BN-centered control view 已经换成更强的 neural baseline
- 当前真正**非 abstain 且非化学合理性拒绝**的正向 follow-up，只剩：
  - `BCN2`
  - `SiBN2`
  - `Si2BN`

因此，这层输出真正补上的不是“更会讲故事”，而是：

**当前系统终于会明确告诉你，什么时候应该继续做，什么时候应该先保守停下。**

同时也必须保留边界：

- 这仍然是 heuristic decision layer；
- 不是 calibrated uncertainty；
- 不是 discovery confidence。

## 6. 当前版本的研究价值应该如何评价

我认为，当前版本的价值主要在以下三点。

#### 6.1 从“工程 demo”推进到“方法学更诚实的研究型 PoC”

旧版本更容易被批评为：

- 有工程流程，但研究边界不清；
- 有候选排序，但 BN 主题不够 central；
- 有结果数字，但对外推边界解释不足。

当前版本至少把这些问题中的大部分都做了显式处理：

- 评估协议更严格；
- grouped robustness 补上了“不是单次 split 运气”的问题；
- BN-slice benchmark 补上了“BN 到底有没有真正被评估”的问题；
- ranking 被拆成 raw / shortlist / BN-centered control / structure follow-up 多层输出；
- 候选不再只是 generic toy space；
- 从 ranking 到结构后续的桥接已经进入主流程。

#### 6.2 当前版本更适合接受导师追问

从导师视角，当前版本最重要的进步不是“分数变高了多少”，而是：

- 能明确区分 overall evaluation、formula-only screening、BN-slice best 这三个不同角色；
- 能明确解释为什么 standard test benchmark 不能代表 BN generalization；
- 能说明为什么 structure-aware 路径只能用于 evaluation，不能直接用于无结构候选；
- 能说明 shortlist 与 structure follow-up artifact 的角色边界。

这会让项目从“看起来什么都想做”变成“知道自己做到了哪里、没做到哪里”。

#### 6.3 项目现在已经有可继续推进的研究骨架

目前代码和 artifact 已经形成一个相对完整的骨架：

- 数据清洗与结构摘要
- 分层评估
- BN-centered 诊断
- 分层候选筛选
- structure follow-up planning

这意味着后续工作不必再从零开始，而可以在已验证框架上继续推进更强的问题，例如：

- BN-centered modeling / decision policy
- 更强的 BN-aware abstention 机制
- 真实的结构生成与松弛工作流
- 更可靠的物理约束或稳定性代理

## 7. 当前最需要诚实承认的不足

这部分我认为必须直接写清，否则整份报告会显得不够可信。

#### 7.1 BN-centered generalization 仍然是当前主要短板

虽然整体 test benchmark 已经不错，但 BN-slice 诊断已经说明：

- selected model 未优于 dummy
- screening model 未优于 dummy
- 只有一个 structure-aware + linear 的组合略优于 dummy

所以当前不能宣称：

- 模型已经学会了 BN 方向的可推广规律；
- 或者当前 ranking 已经能够可靠支持 BN 新材料发现。

#### 7.2 当前 novelty 仍然只是 formula-level

当前项目已经区分了：

- rediscovery
- unseen formula
- follow-up readiness

但它依然只是在**公式层面**处理 novelty，而不是在结构层面或实验层面处理 novelty。

#### 7.3 structure follow-up artifact 已经超过纯 planning，但仍然不是 validation

当前 system 已能输出：

- seed
- handoff
- raw reference records
- job plan
- queue
- candidate-level follow-up shortlist
- first-pass execution JSON / summary / variants / `.cif`

这意味着它已经不只是“把后续工作计划化”，而是已经把一批高优先候选 materialize 成了 deterministic unrelaxed prototype。

但它仍然不能替代：

- 真实结构生成
- 真实 relaxation
- 真正的热力学或动力学判断
