# 项目汇报

## 项目题目

**AI for BN：一个引入轻量 structure-aware evaluation、grouped-by-formula robustness benchmarking、candidate-level grouped-robustness penalty、BN-anchored candidate family generation、BN-local support、BN analog evidence、BN analog validation proxy / penalty、family-aware proposal shortlist、formula-level extrapolation shortlist，以及诚实 formula-only screening fallback 的 band gap 预测 PoC**

---

## 1. 当前结论

本轮工作的核心，不是继续往旧的 toy candidate grid 上堆注释，而是把 candidate-space 设计本身拉回 BN 主题。

当前系统已经完成：

- 使用 JARVIS / 2DMatPedia `twod_matpd` 构建标准化二维材料数据表；
- 采用 grouped-by-formula 的 train / val / test 评估协议；
- 在三条特征路径与两类主模型上做 validation selection；
- 保留 composition-only 控制组，同时保留 lightweight structure-aware evaluation 路径；
- 在测试集上输出完整 benchmark，并保留 `dummy_mean` baseline；
- 在 grouped-by-formula 的 cross-validation 下额外输出 robustness benchmark，避免方法结论只依赖一次 holdout split；
- 再把 grouped robustness 继续接回 candidate ranking，用 grouped-fold prediction spread 做一个轻量 penalty；
- 将 formula-only candidate space 从旧的 III-V toy grid 改为 **BN-anchored formula-family grid**；
- 对该 candidate space 增加 chemical plausibility、domain-support、BN-local support、BN analog evidence、BN analog validation proxy / penalty、novelty layering 与显式 top-k selection decision；
- 在 ranking artifact 中明确区分 best overall evaluation combo 与 best formula-only screening combo；
- 在保留完整 raw ranking 的同时，额外输出 family-aware proposal shortlist 与 formula-level extrapolation shortlist；
- 输出结构化 artifact，并可通过 notebook / Streamlit 展示。

因此，当前项目状态最准确的概括是：

**工程 PoC 成立；评估与筛选的逻辑边界已被显式写清；candidate-space 叙事已经从 generic III-V toy grid 推进到 BN-centered demo family space，而且 BN 已开始进入 screening 逻辑本身、候选解释本身，以及一个轻量 validation-proxy 层，同时候选排序也不再只依赖单次 `train+val` 拟合，并且 advisor-facing 输出已经被拆成 raw ranking、general proposal shortlist 与 extrapolation shortlist 三层。**

但必须强调：

**这仍然不是可信的 structure-aware candidate discovery system，更不是实验候选发现结论。**

## 2. 这轮之后，主要解决了什么问题

### 2.1 评估协议继续保持在 grouped-by-formula

数据集中共有：

- 6351 条记录
- 4381 个唯一 `formula`

因此若按随机行切分，formula leakage 风险会非常明显。

当前默认 split 仍为：

**按 `formula` 分组切分。**

最新验证运行中：

- train：5101 rows / 3505 formulas
- val：618 rows / 438 formulas
- test：632 rows / 438 formulas
- `train_val` / `train_test` / `val_test` overlap 全部为 **0**

这意味着当前 test 指标不再建立在 formula 泄漏之上。

### 2.2 特征空间仍然是“三路径搜索”

当前默认候选特征集为：

1. `basic_formula_composition`
   - 7 维手写基础控制组

2. `matminer_composition`
   - 19 维 composition-only 特征
   - 来自 `Stoichiometry` 与选定 `Magpie` 元素属性统计

3. `matminer_composition_plus_structure_summary`
   - 上述 19 维 + 11 个结构摘要列
   - 结构摘要来自 cached raw JARVIS `atoms` / `lattice` 信息

这里的结构路径仍然是轻量级摘要，不是图网络或高阶结构表征。
但它已经足以让系统保留一个重要的方法学分叉：

- overall evaluation 可以选 structure-aware 路径；
- formula-only candidates 不能。

### 2.3 validation selection 仍然显式区分 evaluation 与 screening

当前验证集选择不是只选模型，而是在以下 6 个组合中选：

- `basic_formula_composition + hist_gradient_boosting`
- `basic_formula_composition + linear_regression`
- `matminer_composition + hist_gradient_boosting`
- `matminer_composition + linear_regression`
- `matminer_composition_plus_structure_summary + hist_gradient_boosting`
- `matminer_composition_plus_structure_summary + linear_regression`

当前结果仍然是：

- **best overall evaluation combo**
  - `matminer_composition_plus_structure_summary + hist_gradient_boosting`
- **best formula-only screening combo**
  - `matminer_composition + hist_gradient_boosting`

两者不同。

而且这一步现在不只由单次 holdout 支撑，也被 grouped robustness 支撑：

- structure-aware best：`MAE_mean = 0.6050 ± 0.0176`
- formula-only best：`MAE_mean = 0.6321 ± 0.0192`

这意味着系统继续避免一个常见错误：

**不会把“测试集上最优的 structure-aware 组合”直接拿去给没有结构的候选公式打分。**

### 2.4 grouped robustness 现在把“不是单次 split 运气”这件事补上了

前面的 grouped train / val / test split 只能说明：

- 当前 test 指标没有 formula leakage；
- 当前 test split 本身是相对诚实的。

但这还不够，因为老师仍然可以质疑：

- 你是不是只是在这一刀切分上恰好表现得不错？

所以这一轮又额外加入了 **grouped-by-formula cross-validation robustness benchmark**。

最新 `robustness_results.csv` 显示：

- `matminer_composition_plus_structure_summary + hist_gradient_boosting`
  - `MAE_mean = 0.6050 ± 0.0176`
- `matminer_composition + hist_gradient_boosting`
  - `MAE_mean = 0.6321 ± 0.0192`
- `basic_formula_composition + hist_gradient_boosting`
  - `MAE_mean = 1.0116 ± 0.0173`
- `dummy_mean`
  - `MAE_mean = 1.1527 ± 0.0233`

这意味着：

- lightweight structure-aware 路径相对最佳 formula-only 路径的优势，
- 在 grouped-by-formula 的 5-fold robustness 里仍然存在，
- 而且它和单次 test benchmark 的方向一致。

所以现在更准确的说法不是“structure-aware 在这一次 test 上更好”，而是：

**structure-aware 路径在当前数据表上，对 grouped-by-formula 的单次 holdout 与多折 robustness 都表现出小幅但一致的优势。**

当然，这仍然不是外部验证，更不是 prospective validation。

### 2.5 grouped robustness 现在也进入 candidate ranking 本身

只把 grouped robustness 放在 evaluation artifact 里，还不够。

因为 candidate ranking 以前仍然主要依赖：

- `train+val` 上的一次拟合；
- 外加一个很小的 feature/model disagreement heuristic。

所以这一轮又额外加入了 **candidate-level grouped robustness penalty**：

- 只使用 `train+val` 参考集，不吃 test labels；
- 用 selected formula-only screening combo 在 grouped-by-formula 的 5 个 folds 上重复训练；
- 对每个 candidate 计算 fold-to-fold prediction spread；
- 再把这个 spread 作为一个轻量 penalty 接回 ranking。

当前 artifact 已显示：

- `grouped_robustness_penalty_weight = 0.15`
- `grouped_robustness_prediction_fold_count = 5`
- `grouped_robustness_prediction_std_mean = 0.3660`
- `grouped_robustness_penalized_rows = 25`

它的含义不是“spread 大就一定是假候选”，而是：

**如果某个公式候选对 grouped training folds 更敏感，那么它的排序就不该和更稳定的候选完全等价。**

目前前列候选里：

- `BCN2` 的 grouped robustness penalty 约为 `0.0897`，仍排第 1；
- `BCN` 的 grouped robustness penalty 约为 `0.1068`，排第 2；
- `BC2N` 虽然是 `reference_like`，但 grouped robustness penalty 约为 `0.1247`，因此排第 3；
- `TlBN2` 的 grouped robustness penalty 只有约 `0.0197`，但仍因 BN-support 与 analog-validation 层被压在更后面。

所以现在 candidate ranking 至少已经开始同时反映：

- predicted band gap；
- 小模型池 disagreement；
- train+val feature-space support；
- BN-local support；
- BN analog validation；
- 以及 grouped-fold split robustness。

### 2.6 candidate space 已从旧 III-V toy grid 迁移到 BN-anchored family grid

这是本轮最核心的升级。

当前默认 candidate generation strategy 为：

- `bn_anchored_formula_family_grid`

其基本结构是：

- `BN` 二元 anchor
- Group-IV BN ternary families
  - `group14_bn_111_family`
  - `group14_bn_211_family`
  - `group14_bn_121_family`
- Group-III BN ternary families
  - `group13_bn_111_family`
  - `group13_bn_211_family`
  - `group13_bn_121_family`

这个设计不是随意改名，而是把 candidate space 锚定到三类更像 BN 研究叙事的母题：

- `BN`
- `BCN / h-BCN`
- `BC2N`
- `Si2BN` 一类数据集内真实出现过的 BN ternary motif

因此，现在候选空间终于不再像“先有 generic III-V grid，再硬挂一个 BN 标签”，而是：

**candidate-space 本身就围绕 BN-containing formula families 组织。**

### 2.7 candidate provenance 现在可以在 artifact 中直接追踪

当前 ranking artifact 不只给出公式和分数，也会显式保留 candidate provenance，例如：

- `candidate_generation_strategy`
- `candidate_space_name`
- `candidate_space_kind`
- `candidate_family`
- `candidate_template`
- `candidate_family_note`

当前 family 计数为：

- `bn_binary_anchor`: 1
- `group14_bn_111_family`: 4
- `group14_bn_211_family`: 4
- `group14_bn_121_family`: 4
- `group13_bn_111_family`: 4
- `group13_bn_211_family`: 4
- `group13_bn_121_family`: 4

因此，现在不仅能看“哪个公式排前面”，还能看：

**它到底是从哪一种 BN family 假设里生成出来的。**

### 2.8 chemical plausibility 仍然是轻量层，但对象已经换成 BN family candidates

本轮没有假装解决稳定性、结构或可合成性问题，而是继续使用一层**轻量、可解释、公式级**的 chemical plausibility 检查：

- 方法：`pymatgen` oxidation-state guesses
- 目标：检查候选公式是否存在基础的 charge-balanced common oxidation-state assignment

在新的 BN-anchored candidate space 上，结果为：

- **23 / 25** 个候选通过
- **2 / 25** 个候选失败
- 当前失败公式：
  - `AlBN`
  - `TlBN`

这意味着 chemical plausibility 现在仍然只是 credibility boundary，不是 discovery filter。

### 2.9 novelty layering 现在终于不再只是形式上的诚实

上一轮的 novelty layer 已经能区分：

- `train_plus_val_rediscovery`
- `held_out_known_formula`
- `formula_level_extrapolation`

但旧 candidate grid 的问题是：

- 大多数候选本来就在 dataset 中出现过；
- novelty layer 虽然诚实，但结果本身并不强。

而在新的 BN-anchored candidate space 中，最新 artifact 的真实计数为：

- `train_plus_val_rediscovery`：**3**
- `held_out_known_formula`：**0**
- `formula_level_extrapolation`：**22**

其中 rediscovery 具体是：

- `BN`
- `BC2N`
- `Si2BN`

标准 top-k 20 的构成为：

- rediscovery：**3**
- held-out-known：**0**
- formula-level extrapolation：**17**

因此，这轮 candidate-space redesign 带来了一个真实变化：

**ranking 的主叙事已经从“已知公式重放”切换成“以 formula-level extrapolation 为主的 BN-family demo screening”。**

当然，这里必须再强调一次：

- formula-level extrapolation 只是“整个 dataset 没见过这个化学式”
- 并不等于结构稳定、可合成或实验可实现

### 2.10 现在不只看全局 domain-support，还看 BN-local support、BN analog evidence 与 analog-validation penalty

上一轮 domain-support 的问题是：

- 它只能回答候选是否偏离整个 train+val 数据分布
- 但还不能回答候选是否偏离已知 BN slice 本身

因此这一轮先加入了 **BN-local support** 层，直接用 train+val 中已知 BN-containing formulas 做局部参考。

当前 artifact 中：

- `domain_support_penalized_rows = 12`
- `domain_support_low_support_rows = 12`
- `bn_support_reference_formula_count = 10`
- `bn_support_penalized_rows = 15`
- `bn_support_low_support_rows = 15`

这意味着 ranking 现在不只受“全局公式空间支持度”影响，还会受“是否靠近已知 BN 公式族”影响。

例如当前前列候选已经会显式写出 BN-local 最近锚点：

- `BCN2 -> B2(CN2)3`
- `BCN -> BC2N`
- `AlBN2 -> Si2BN`

但这还不够，所以这一轮又加了 **BN analog evidence** 层。

它不是去假装预测 unseen candidate 的真实稳定性，而是去检索这些候选附近的 BN analog 在真实数据里的观测属性：

- `band gap`
- `energy_per_atom`
- `exfoliation_energy_per_atom`
- `total_magnetization`

当前 summary 中：

- `bn_analog_evidence_enabled = True`
- `bn_analog_reference_formula_count = 10`
- `bn_analog_reference_exfoliation_energy_median = 0.07173938218750031`
- `bn_analog_reference_energy_per_atom_median = -7.929051850789474`
- `bn_analog_reference_abs_total_magnetization_median = 0.0`
- `bn_analog_exfoliation_available_rows = 25`
- `bn_analog_lower_or_equal_reference_rows = 2`
- `bn_analog_higher_reference_rows = 23`
- `bn_analog_validation_penalty_weight = 0.12`
- `bn_analog_validation_penalized_rows = 23`

例如：

- `BC2N` 的邻近 BN analog exfoliation mean 低于 BN reference median
- `BCN2 / BCN / AlBN2` 的邻近 BN analog exfoliation mean 高于该 median

在此基础上，这一轮又把 analog 证据再压缩成一个更可读的 **BN analog validation proxy**，看这些邻近 BN analog 在 `exfoliation / energy / magnetization` 三个可用指标上，是否整体更接近 BN 参考区间：

- `reference_like_on_available_metrics`: `2`
- `mixed_reference_alignment`: `2`
- `reference_divergent_on_available_metrics`: `21`

前列候选中的典型例子是：

- `BC2N` 属于 `reference_like`
- `BCN2 / BCN` 属于 `mixed`
- `AlBN2 / SiBN2 / SiBN` 属于 `reference_divergent`

更重要的是，这个 validation proxy 现在已经不只是说明字段，而是开始进入 ranking：

- `BCN2 / BCN` 仍排前二，但各自吃到轻量 mixed penalty
- `BC2N` 现在升到第 `3`，且属于 `reference_like`
- `AlBN2 / SiBN2 / SiBN` 这类 `reference_divergent` 候选会再被轻量压分

因此，现在可以更严谨地说：

**BN 已不只是 candidate-space 命名主题，而是 screening 里的显式参考层、analog-evidence 层，以及一个轻量 validation-proxy / ranking-penalty 层。**

### 2.11 现在不只输出 raw ranking，还显式拆成两个 advisor-facing shortlist

如果只保留 raw top-k，一个仍然存在的问题是：

- 虽然当前 top-k 已经以 `formula_level_extrapolation` 为主；
- 但汇报时仍然容易混入 rediscovery；
- 也仍然容易被少数 `candidate_family` 重复占据太多位置。

因此这一轮又增加了两层与 raw ranking 分离的输出：

1. **`demo_candidate_proposal_shortlist.csv`**
   - 面向一般汇报 / 跟进场景；
   - 保留 chemical plausibility 优先；
   - 按 raw ranking 顺序行走；
   - 对单一 `candidate_family` 施加 family cap。

   当前配置与结果：
   - `shortlist_size = 10`
   - `max_per_candidate_family = 2`
   - `proposal_shortlist_selected_rows = 10`
   - `proposal_shortlist_novelty_bucket_counts = {train_plus_val_rediscovery: 3, held_out_known_formula: 0, formula_level_extrapolation: 7}`

2. **`demo_candidate_extrapolation_shortlist.csv`**
   - 只保留 `formula_level_extrapolation` 候选；
   - 继续保留 chemical plausibility 优先；
   - 继续做 family cap，让 shortlist 更像一个 discovery-style follow-up list。

   当前配置与结果：
   - `shortlist_size = 5`
   - `max_per_candidate_family = 1`
   - `extrapolation_shortlist_candidate_count = 22`
   - `extrapolation_shortlist_selected_rows = 5`
   - `extrapolation_shortlist_selected_family_counts = {group14_bn_121_family: 1, group14_bn_111_family: 1, group13_bn_121_family: 1, group13_bn_211_family: 1, group14_bn_211_family: 1}`
   - 当前 shortlist 为：`BCN2`, `BCN`, `AlBN2`, `Al2BN`, `Ge2BN`

这一步的意义不是把 raw ranking 改写成“看起来更漂亮”的结果，而是把三种不同职责拆开：

- **raw ranking**：保留完整 source-space 排序，保证方法学诚实；
- **proposal shortlist**：给一般汇报场景一个不那么家族扎堆的列表；
- **extrapolation shortlist**：给真正想继续追踪未见公式的人一个不被 rediscovery 干扰的起点。

这直接回应了前面 novelty layer 仍可能在汇报上被误读的问题，也让当前 artifact 更接近博士阶段真正会拿去讨论下一步验证对象的样子。

## 3. 当前系统设定

### 3.1 数据与任务

- 数据集：`twod_matpd`
- 数据源：JARVIS / 2DMatPedia
- 目标列：`band_gap`
- 总数据量：6351 rows
- 唯一 `formula`：4381
- BN 主题切片：12 rows / 10 unique BN formulas

BN 当前仍然主要是：

- 主题切口
- 候选空间组织中心
- 结果展示焦点

训练主体仍然是整个二维材料表，而不是 BN-only 学习。

### 3.2 模型流程

当前默认流程为：

1. 构造 grouped split；
2. 构造三套 feature tables；
3. 在 validation 上选择 best overall evaluation combo；
4. 同时选出 best formula-only screening combo；
5. 用 `train + val` 重训 overall best combo，并在 test 上评估；
6. 对所有组合与 `dummy_mean` 生成 benchmark；
7. 运行 grouped-by-formula robustness benchmark；
8. 再运行专门的 BN-focused leave-one-BN-formula-out benchmark；
9. 用公式兼容的 screening combo 生成 BN-family candidate ranking；
10. 将 honesty-oriented metadata 写入 artifact。

这部分结果非常重要，因为它把 baseline 问题和 BN 中心性问题都正式摆到了台面上。

### 4.5 对这些数字的克制解读

可以成立的结论有四点：

1. 在普通 test split 上，lightweight structure-aware evaluation 相对最佳 formula-only 路径仍有**小幅但一致**的提升。
2. richer composition representation 明显优于最弱的手写控制组。
3. 在 BN-focused benchmark 上，当前 overall selected model 与 formula-only screening model 都**没有打赢**最基本的 global dummy baseline，这说明现有“整体最优”不能被直接当成“BN 最优”。
4. 但 BN-slice best configured combo 已经变成 `matminer_composition_plus_structure_summary + linear_regression`，而且它确实略优于 dummy，这说明项目不是完全没有 BN-centered signal，只是这一点还没有成为当前主选择路径。

但不能夸大的地方同样明确：

- 这不是成熟 structure-aware 建模；
- 这不足以支撑 discovery claim；
- 候选 discovery credibility 的改善，主要来自 **candidate-space redesign**，不是预测分数突然飞跃；
- 当前必须把 **overall evaluation best / formula-only screening best / BN-focused benchmark best** 作为三个不同角色来汇报，而不是混成一句“当前最优模型”。

### 4.6 当前 demo candidate ranking 的解读

当前前列候选已经明显是 BN-family 风格，例如：

但这仍不等于：

- 这些候选已经具备可信的材料发现价值。

## 5. 当前方法学判断

### 5.1 已经站得住的部分

- grouped-by-formula 评估协议；
- feature/model benchmark；
- cached raw JARVIS 数据驱动的轻量 structure-aware evaluation 路径；
- overall evaluation 与 formula-only screening 的显式分离；
- BN-anchored candidate family generation；
- ranking 中的 chemical plausibility、domain-support、BN-local support、BN analog evidence、BN analog validation proxy、grouped candidate robustness 与 novelty layering；
- 以及单次 holdout benchmark 之外的 grouped-by-formula robustness benchmark；
- 候选 family provenance 的显式暴露；
- 与 raw ranking 分离的 proposal shortlist 与 extrapolation shortlist。

### 5.2 仍然站不住的部分

- 结构路径仍只是摘要级，不是强结构表征；
- formula-only candidates 仍然无法进行真正的 structure-aware screening；
- `22 / 25` 的未见候选只是 formula-level extrapolation，不是 structure-level discovery；
- proposal shortlist 与 extrapolation shortlist 只是更诚实的汇报 / 跟进选择层，不是结构验证、稳定性筛选或实验优先级证明；
- BN slice 太小，无法支撑“面向 BN 的方法优势”结论；
- 当前 candidate space 仍无结构、稳定性、可合成性约束；
- disagreement、grouped candidate robustness、domain-support、BN-local support、BN analog evidence 与 BN analog validation proxy / penalty 仍然不是直接的 candidate stability proof，也不是 calibrated uncertainty；
- grouped robustness 比单次 split 更好，但仍然只是同一数据表内的 grouped cross-validation，不是外部数据验证；
- train / inference 仍未解耦。

因此，此阶段最准确的评价应当是：

**PoC 已经从“诚实的 formula-only ranking demo”进一步推进到了“BN-anchored candidate-space demo with grouped robustness support, candidate-level grouped-robustness penalty, BN-local support, BN analog evidence, BN analog validation proxy / penalty, honest formula-only screening fallback, and split advisor-facing shortlist layers”。**

这是一种更好的博士级 PoC 形态，但仍不是研究结论本身。
